# Changelog — 18 de julio de 2026

## `37de137` — `fix: make member notifications responsive and actionable`

- Se corrigió el solapamiento de las notificaciones en dispositivos móviles aislando las dimensiones de las tarjetas frente a estilos globales de Webflow.
- Se eliminó el parpadeo periódico: el preview deja de hacer polling y producción solo repinta la campana cuando cambian los datos, con consultas cada 15 segundos.
- Se añadió un resolvedor seguro de acciones que abre el expediente y chat de una mascota, el detalle interno de un reintegro o un modal informativo según el contexto.
- Se conservaron las notificaciones históricas mediante traducción de enlaces antiguos y se bloquearon destinos externos no autorizados.
- Las notificaciones nuevas de solicitudes de información, apelaciones, estados de mascotas y Fondo Solidario ahora incluyen metadata canónica.
- Se añadieron pruebas de responsive, seguridad, compatibilidad histórica, productores backend y estabilidad del polling.

### Impacto funcional

Los miembros pueden leer las notificaciones correctamente en celular y acceder directamente a la acción relacionada sin salir innecesariamente del dashboard unificado. Las actualizaciones periódicas ya no generan recargas visuales cuando no existen novedades.

## `67422b6` — `feat: implementar bajas revisables y ciclo solidario anual`

- Las solicitudes de baja de mascotas ahora quedan pendientes y mantienen ocupado el espacio hasta la aprobación administrativa.
- Se añadió al dashboard administrativo una cola autenticada para aprobar o rechazar bajas, con notificación al miembro.
- El widget unificado muestra el estado “Baja solicitada” y evita duplicar solicitudes mientras continúa la revisión.
- El saldo del Fondo Solidario ahora se calcula por el aniversario del primer pago de cada membresía, incluyendo años bisiestos.
- Las tarjetas muestran la fecha real de la próxima renovación anual del saldo.
- El formulario de reintegros muestra el aviso de fondos insuficientes únicamente cuando el monto solicitado supera el disponible del fondo seleccionado.
- El backend valida nuevamente el ciclo, elegibilidad y saldo, y devuelve el código `INSUFFICIENT_SOLIDARITY_BALANCE` cuando corresponde.
- Se agregó la migración `20260718_pet_unsubscription_workflow_and_solidarity_cycle.sql`, aplicada antes del despliegue.

### Impacto funcional

Las bajas de peludos ya requieren una decisión del comité antes de liberar lugares de la manada. Los reintegros utilizan el ciclo anual real de cada miembro y bloquean solicitudes que exceden el saldo disponible con un mensaje contextual.
