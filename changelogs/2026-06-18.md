# Changelog 2026-06-18

## ✨ Features

### Wellness Leads - Checkboxes + Campo "Otro" + Vista Detallada en Admin

Se ha actualizado completamente el flujo de registro de leads de centros de bienestar (wellness leads) para soportar selección múltiple de servicios mediante checkboxes, campo libre "Otro" y vista detallada en el dashboard de administración.

#### Cambios implementados:

1. **Endpoint Webflow** (`src/app/api/webflow/wellness-lead/route.ts`)
   - Maneja `services` como **array** (checkboxes múltiples nativos del FormData) o string separado por comas (compatibilidad)
   - Nuevo soporte para campo `other_service`: si el usuario marca "Otro" y escribe texto libre, se guarda en BD como `Otro: <texto personalizado>`
   - Aplica tanto en creación de nuevos leads como en actualización de leads duplicados existentes

2. **Endpoint Admin** (`src/app/api/admin/wellness-leads/route.ts`)
   - Nuevo parámetro `?id=<uuid>` para obtener un solo lead (GET by ID)
   - Usado por el modal de detalle en la tabla
   - Mantiene exportación CSV/JSON funcionando sin cambios

3. **Tabla Admin / Dashboard** (`src/components/Admin/WellnessLeadsTable.tsx` + `WellnessLeadsTable.module.css`)
   - **Filas clickeables** → abren **modal con toda la información organizada en secciones**:
     - 📋 Información del Establecimiento (nombre, servicios, dirección completa, mascotas estimadas/mes)
     - 👤 Contacto (nombre, rol, email, teléfono, WhatsApp)
     - 🌐 Redes Sociales y Web (sitio web, Instagram, Facebook, TikTok)
     - 🏷️ Servicios Disponibles (tags coloreados: Veterinaria 🏥, Grooming ✂️, Hotel 🏨, Tienda 🛍️)
     - 📝 Descripción (si existe)
     - 📊 Información del Lead (fuente, **estado editable via dropdown**, score, **asignado editable via dropdown**, fechas de creación/actualización)
     - 📌 Notas (si existen)
   - Animaciones suaves: `fadeIn` (overlay) + `slideUp` (modal)
   - Scroll personalizado con colores de marca
   - Header sticky en modal para navegación larga

4. **Formulario Webflow** (`public/wellness-lead-form.html`)
   - Checkbox **"Otro"** agregado a la lista de 6 tipos de establecimiento
   - Campo de texto condicional ("Especificar otro tipo") que aparece/oculta dinámicamente al marcar/desmarcar "Otro"
   - Validación: si marca "Otro", el campo de texto es requerido
   - Envía `services` como array nativo (FormData con múltiples valores mismo nombre) + `other_service` separado
   - Formato de teléfono auto-formateo (XXX XXX XXXX)
