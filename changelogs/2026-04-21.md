# Changelog - 2026-04-21

## Resumen de Cambios
1. **Sincronización de Pasos con URL**: Implementación de un sistema de navegación basado en parámetros de URL (`?step=X`) para el flujo de registro.
2. **Mejora de Resiliencia en Navegación**: Soporte completo para el botón "Atrás" del navegador y corrección de regresiones accidentales al Paso 2.
3. **Optimización de Retorno desde Stripe**: Configuración de `cancelUrl` para asegurar un re-aterrizaje preciso en la selección de plan.

## Navegación y UX

### src/components/RegistrationV2/NewRegistrationFlow.tsx [MODIFY]
- **Gestión de Estado**: El estado `currentStep` ahora se inicializa como `null` para permitir una carga inicial limpia sincronizada con la URL.
- **Helper `goToStep`**: Nueva función que unifica el cambio de estado con `window.history.pushState` (o `replaceState`), garantizando que la URL (`?step=X`) siempre refleje el progreso real.
- **Listener `popstate`**: Implementación de escucha activa del historial del navegador. Ahora, al presionar "Atrás", el componente retrocede de paso visualmente sin recargas innecesarias.
- **Prioridad de Inicialización**: Se modificó `loadSavedState` para dar prioridad al paso detectado en la URL si el usuario ya tiene sesión activa. Esto previene que la latencia de la base de datos o Memberstack regrese al usuario al Paso 2 cuando ya estaba en el Paso 3.
- **Stripe Checkout**: Se añadió la propiedad `cancelUrl: window.location.href` a la llamada de `purchasePlansWithCheckout`. Esto garantiza que si el usuario decide no pagar en ese momento y presiona "Atrás" en Stripe, regrese exactamente al Paso 3 de selección de plan.

## Impacto Funcional
- **Eliminación de Bucles de Navegación**: Se resolvió el error reportado donde en dispositivos iPhone (y otros navegadores) el usuario era enviado al Paso 2 al intentar regresar desde la pasarela de pago.
- **Navegación Intuitiva**: El registro ahora se comporta como una aplicación multi-página estándar para el navegador, permitiendo usar las flechas laterales de forma predecible.
- **Recuperación de Sesión**: Mayor robustez al retomar registros pendientes desde links directos.

## Verificación Técnica
