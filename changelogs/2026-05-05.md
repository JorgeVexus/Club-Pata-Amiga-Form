# Changelog - 2026-05-05

Hoy se ha implementado el soporte completo para el registro de múltiples mascotas (hasta 3) desde el inicio del flujo de registro (Paso 2), y se ha rediseñado por completo la interfaz de estatus "Rechazado" en el Widget Unificado con una estética premium y soporte para navegación entre múltiples mascotas.

## Cambios Realizados

### [Registration Flow]
- **Refactorización de Estado**: Se cambió el estado `petBasic` de un objeto único a un array en `NewRegistrationFlow.tsx`.
- **Sincronización con Memberstack**: Se implementó el mapeo dinámico de campos indexados (`pet-1-name`, `pet-2-type`, etc.) para asegurar que hasta 3 mascotas se guarden correctamente en los custom fields.
- **Recuperación de Datos**: Se actualizó la lógica de carga inicial para reconstruir el array de mascotas desde Memberstack o `localStorage` en caso de recarga de página.
- **Cálculo de Carencia**: Se ajustó el cálculo de períodos de carencia para aplicarse a cada mascota individualmente si se usa un código de referido.

### [Step 2: Datos Básicos]
- **Interfaz Multimascota**: Se añadió un botón "+" para permitir añadir hasta 2 mascotas adicionales (máximo 3 en total).
- **Diseño Responsivo**: Se ajustó el layout a vertical para los campos de edad y tipo, evitando desbordamientos en móviles.
- **Validación Dinámica**: Se implementó validación independiente para cada mascota agregada.
- **FIX: Campo de Edad**: Se corrigió un error donde el campo de edad no permitía escribir debido a actualizaciones de estado asíncronas competitivas. Se refactorizó `updatePet` para usar actualizaciones funcionales y se mejoró la robustez de `AgeInput`.

### [Step 5: Información Complementaria]
- **Iteración de Mascotas**: El componente ahora renderiza secciones independientes para cada mascota registrada en el Paso 2.
- **Visual Feedback**: Se mejoró el estilo de los botones de raza seleccionados (ahora con fondo naranja y borde marcado).
- **Lógica Senior**: Se añadió el mensaje personalizado para mascotas de 10+ años, indicando la posibilidad de subir el certificado médico en el momento o después.
- **Campos de Adopción**: Se incluyeron los campos de "Es adoptado" e historia de adopción para cada mascota.
- **UI de Separación**: Se añadieron divisores y badges con iconos (🐶/🐱) para diferenciar claramente a los peludos de la manada.

### [Estilos y UI]
- **Step5CompletePet.module.css**: Se añadieron nuevos estilos para `.petSection`, `.petBadge`, `.divider` y se refinó `.switchActive`.
- **Step2PetBasic.module.css**: Se añadieron estilos para las tarjetas de mascota adicionales y botones de gestión.

### [Unified Membership Widget]
- **Rediseño Premium de Estatus Rechazado**: Implementación visual de alta fidelidad para el estado "Rejected" con fondo naranja (#FE8F15), patrón de marca "A" al 20% de opacidad y tarjeta blanca con bordes de 66px.
- **Soporte Multimascota en Rechazado**: Añadida una barra de pestañas dinámica para que usuarios con múltiples mascotas puedan ver el estatus de cada una de forma independiente en la vista premium.
- **Integración de Chat de Apelación**: Se corrigió la lógica de vinculación de eventos para asegurar que el botón de apelación abra el modal de historial de comunicación y chat de soporte.
- **Herramientas de Debug**: Se registró el objeto `window.pataDebug` en el entorno global para facilitar la validación de estados de membresía (Rechazado, Aprobado, Pendiente) desde la consola.
- **Correcciones de UI**: Ajustes de responsividad para mobile (stack vertical) y visuales (rotación de imagen de mascota y sombras).
