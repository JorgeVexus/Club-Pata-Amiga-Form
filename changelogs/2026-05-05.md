# Changelog - 2026-05-05

## 📊 Refinamiento del Dashboard Admin y Herramientas de Desarrollo Local

Se han implementado mejoras significativas en el monitoreo administrativo de mascotas y se ha creado un entorno de pruebas local para el Widget de Membresía Unificado.

### 📋 Cambios Principales

#### 🖥️ Dashboard Admin (Gestión de Mascotas)
- **Estandarización de Alerta de Mascotas**:
  - Se cambió la etiqueta de "X por aprobar" a **"X por revisar"** para reflejar mejor la tarea administrativa.
  - **Lógica de Conteo Ampliada**: Ahora el indicador cuenta todas las mascotas que no están en estado `approved`, incluyendo estados de `pending`, `action_required`, `appealed` y `rejected`.
  - **Visibilidad Inteligente**: La alerta se muestra específicamente en miembros aprobados que aún tienen mascotas pendientes de revisión final, evitando que pasen desapercibidas.
- **Optimización de API**: Refactorización del endpoint `/api/admin/members` para calcular el conteo basado en la exclusión del estado aprobado.

#### 🎡 Unified Membership Widget (Entorno Local)
- **Herramienta de Debug Local (`dev-test.html`)**:
  - Creación de una página de prueba independiente que permite previsualizar el widget sin despliegues a producción.
  - **Simulador de Estados**: Panel interactivo integrado para forzar el widget a mostrar estados de `aprobado`, `rechazado`, `pendiente`, `acción requerida` y `apelado`.
  - Facilita el ajuste de diseño y lógica de interacción de forma inmediata.

#### ✅ Control de Calidad (QA)
- **Build Verification**: El proyecto compila correctamente (`npm run build`).
- **Type Checking**: Verificación de tipos exitosa en todos los archivos modificados.

---
**Impacto funcional**: Los administradores tienen ahora una visión más precisa de la carga de trabajo pendiente a nivel de mascotas, y el equipo de desarrollo cuenta con un ciclo de feedback mucho más rápido para mejoras visuales en el widget.
