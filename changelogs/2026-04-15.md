# Changelog - 15 de Abril, 2026

## Resumen de la Sesión
Hoy nos enfocamos en alcanzar la paridad total entre el widget de registro de mascotas (`pet-cards-widget.js`) y el paso 5 del flujo de registro principal. Se implementaron mejoras críticas en UX, validaciones de seguridad (edad mínima y senior) y un sistema de autocompletado inteligente para características físicas de las mascotas.

---

## [1.5.0] - 2026-04-15

### ✨ Características Principales
- **Autocompletado de Colores (Paridad Paso 5)**:
  - Implementación de campos inteligentes para **Color de Pelo**, **Nariz** y **Ojos**.
  - Los datos se cargan dinámicamente desde Supabase filtrados por tipo de mascota (Perro/Gato).
  - Interfaz premium de sugerencias con soporte para selección táctil y de ratón.
- **Selector de Edad Premium**:
  - Migración de inputs numéricos a selectores de botones consistentes con el diseño del paso 2 del registro.
  - Lógica automática para años y meses.
- **Validación de RUAC Oficial**: Implementación de restricción estricta de 11 caracteres para el campo RUAC, alineado con los requerimientos del sistema.

### 🛡️ Validaciones y Lógica de Negocio
- **Lógica de Mascotas Senior Unificada**: Se unificó la lógica de detección senior (10+ años) en todo el sistema. Si una mascota es senior, se activa automáticamente un aviso visual indicando que se requerirá un certificado médico.
- **Validación de Edad Mínima**: Se bloquea el registro de cachorros menores de 4 meses, mostrando el mensaje de alerta correspondiente al flujo oficial.
- **Sincronización Raza/Adopción**:
  - Al seleccionar la opción de **Raza**, el widget ahora limpia automáticamente cualquier historia de adopción previa y resetea el campo de raza para forzar una selección válida.
  - Esto garantiza la integridad de los datos (la historia de adopción solo aplica a mestizos).
- **Aviso Legal de Adopción**: Implementación de un mensaje estático simplificado para la historia de adopción, alineado con el diseño final.

### ⚙️ Mejoras Técnicas (Backend & API)
- **API de Catálogos (`/api/catalogs/coat-colors`)**:
  - Refactorización para soportar múltiples categorías mediante el parámetro `category` (`coat`, `nose`, `eye`).
  - Implementación de headers **CORS** (`Access-Control-Allow-Origin: *`) para permitir la comunicación fluida entre el widget alojado en Webflow y el backend de Next.js.
  - **Hotfix**: Corrección de error de compilación en Vercel relacionado con el *scope* de `searchParams` en el bloque de excepciones.
- **Widget de Notificaciones**: Corrección de problemas de redirección relacionados con el dominio para asegurar que las notificaciones dirijan correctamente al usuario según el entorno (producción/Vercel).
- **Optimización de Caché**: Los catálogos de razas y colores ahora se inicializan una sola vez por flujo de "Añadir Mascota", eliminando peticiones redundantes al servidor.
