# Changelog - 2026-05-07

## 🚀 Mejoras y Correcciones (User Profile API)

### 🛠️ API & Backend
- **Corrección Crítica (SQL Error 42703)**: Se eliminaron las referencias a las columnas legacy `street`, `ext_number` e `int_number` en los endpoints de perfil.
  - Modificado: `/api/user/profile` (lectura)
  - Modificado: `/api/user/update-profile` (escritura)
- **Corrección de Embajadores**: Se eliminó la referencia a la columna inexistente `full_name` en el endpoint `/api/ambassadors/by-memberstack`, reemplazándola por los campos correctos `first_name` y `paternal_surname`.
- **Corrección en Desactivación**: Se eliminó el campo `cancelled_at` en el endpoint `/api/user/deactivate` ya que no existe en el esquema de la base de datos, evitando errores al cancelar membresías.

### 🧩 Widgets (Webflow)
- **User Profile Widget**:
  - Se validó el mapeo de campos para mostrar correctamente Nombre, Teléfono, Fecha de Nacimiento y Dirección consolidada.
  - Se corrigió el fallback del nombre que mostraba el prefijo del correo cuando la API de perfil fallaba debido a los errores de SQL mencionados.
  - Mejorada la visualización de la fecha de próximo pago integrando datos directos de Stripe.

### 👤 Impacto funcional
- Los usuarios ahora pueden ver y editar su información personal sin errores de servidor.
- El dashboard de administración y el widget de perfil son consistentes con el esquema actual de Supabase.
