# Changelog - 2026-05-07

## Resumen de Cambios
Hoy se realizaron optimizaciones críticas en la interfaz móvil del Unified Membership Widget y se corrigió un error de usabilidad en el sistema de notificaciones en tiempo real.

---

### [Unified Membership Widget] - Optimización Móvil
- **Layout Compacto**: Se reestructuró la cuadrícula de beneficios para mostrar 2 columnas en móvil, reduciendo el scroll vertical.
- **Iconografía**: Se ajustó el tamaño de los iconos de beneficios a 20x20px en dispositivos móviles.
- **Espaciado**: Se eliminó el padding del contenedor `.test-widget-user` en móvil para maximizar el uso de pantalla.
- **Textos**: Se simplificaron los textos de los beneficios en la vista "En Revisión".

### [Real-time Notifications] - Corrección de Modal en Móvil
- **Visibilidad**: Se corrigió un error donde el dropdown no aparecía en móvil debido a conflictos de posicionamiento (`top` y `bottom` simultáneos).
- **Responsividad Táctil**: Se añadió soporte para eventos `touchstart` en el icono de la campana para asegurar una apertura inmediata en dispositivos táctiles.
- **Z-Index**: Se forzó un `z-index` máximo y visibilidad absoluta en móvil para evitar que elementos del sitio (como navbars de Webflow) bloqueen el modal.

### [Notification Bell] - Mantenimiento
- **Sincronización**: Se añadieron los mismos listeners de `touchstart` para mantener la consistencia entre los diferentes widgets de notificaciones.

---

### [Unified Membership Widget] - Personalización de Mascotas
- **Fotos en Tabs**: Se implementó el renderizado de la foto real de la mascota en los tabs del dashboard.
- **Respaldo de Iconos**: Se añadieron nuevos iconos personalizados (`cat-icon.svg`, `dog-icon.svg`) como fallback cuando no hay foto disponible.
- **Mejoras Visuales**: Se ajustó el CSS para asegurar que las fotos sean circulares y tengan un encuadre correcto (`object-fit: cover`).
- **Corrección de Iconos**: Se corrigió un error donde los iconos de fallback no se mostraban debido a rutas relativas y diferencias de mayúsculas/minúsculas en el servidor. Ahora se utiliza explícitamente el dominio de producción `https://app.pataamiga.mx` y la carpeta `Icons/` (con I mayúscula) para asegurar compatibilidad total con el servidor de producción.

---

### [Pet Cards Widget] - Mejora de Información General
- **Iconografía Personalizada**: Se reemplazaron los iconos genéricos de Material Symbols por iconos PNG personalizados que describen visualmente cada campo de información general (Especie, Edad, Género, Colores, Ingreso, Activación).
- **Consistencia Visual**: Se ajustó el CSS de los iconos en el modal de detalles para mantener un tamaño uniforme y centrado (22x22px).
- **Robustez de Assets**: Se renombraron los archivos de iconos para eliminar espacios y caracteres especiales, garantizando que las URLs sean seguras y compatibles con el servidor de producción.
- **URLs de Producción**: Se implementó el uso de rutas absolutas (`https://app.pataamiga.mx/Icons/`) para asegurar que los activos carguen correctamente independientemente de donde se use el widget (Webflow, Vercel, etc.).

### [User Settings Widget] - Simplificación Legal y Modal de Términos
- **Unificación Legal**: Se reemplazaron los 4 enlaces PDF individuales por un único acceso "Ver términos y condiciones" con iconografía de documento.
- **Modal de Términos**: Se implementó un modal interactivo que carga dinámicamente los términos legales desde la API (`/api/legal/terms`).
- **Diseño Premium**: El modal sigue la línea estética Neo-Brutalista con overlay desenfocado, bordes de 2px y spinner de carga personalizado.
- **Corrección de Bug**: Se corrigió una referencia inválida a `memberstackUser` en la función de desactivación de cuenta, asegurando que el proceso de cancelación funcione correctamente.

---

### [Solidarity Fund Dashboard] - Implementación y Refinamiento Premium
- **Nueva Funcionalidad**: Implementación inicial del widget de Fondo Solidario para la gestión de apoyos en centros aliados y reembolsos.
- **Diseño Neo-Brutalista**: Aplicación estricta del sistema de diseño de Pata Amiga con bordes de 2px, sombras sólidas y tipografía premium (Fraiche 20px, Outfit 20px).
- **Optimización Móvil**: Reestructuración completa del layout para pantallas pequeñas, incluyendo el uso de `box-sizing: border-box` global para prevenir desbordamientos en buscadores y filtros.
- **Accesibilidad (A11y)**: Se implementó semántica HTML real (`<ul>`/`<li>`), roles de botón, y soporte completo para navegación por teclado (`Enter`/`Espacio`).
- **Design Spells (Animaciones)**: Se añadieron animaciones de entrada escalonadas (*staggered fade-in*) y efectos de hover dinámicos que elevan las tarjetas para una experiencia fluida.
- **Iconografía Dinámica**: Sistema de badges con iconos locales (`/Icons/`) que cambian según el tipo de apoyo (Emergencia, Vacuna, Fallecimiento) y estado de la solicitud.

---

### [Solidarity Fund Dashboard] - Refactorización de Seguridad (Arquitectura API-First)
- **Eliminación de Supabase en Cliente**: Se eliminó la dependencia de `supabase-js` y la exposición de llaves API en el navegador.
- **Migración a API Routes**: El widget ahora consume exclusivamente endpoints de Next.js (`/api/solidarity/*`), protegiendo las credenciales en el servidor (Vercel).
- **Configuración Dinámica**: Se implementó soporte para `window.PATA_AMIGA_CONFIG`, permitiendo una integración limpia con Webflow sin filtrar variables de entorno.
- **Corrección de Error**: Resuelto el mensaje "supabaseUrl is required" al centralizar la autenticación y el acceso a datos en el backend.
- **Arquitectura Mandataria**: Se estableció esta estructura como el estándar obligatorio para todos los futuros widgets del proyecto en `AGENTS.md`.
