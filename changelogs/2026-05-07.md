# Changelog - 2026-05-07

## 🚀 Mejoras y Correcciones (User Profile API)

### 🛠️ API & Backend
- **Corrección Crítica (SQL Error 42703)**: Se eliminaron las referencias a las columnas legacy `street`, `ext_number` e `int_number` en los endpoints de perfil.
  - Modificado: `/api/user/profile` (lectura)
  - Modificado: `/api/user/update-profile` (escritura)
- **Corrección de Embajadores**: Se eliminó la referencia a la columna inexistente `full_name` en el endpoint `/api/ambassadors/by-memberstack`, reemplazándola por los campos correctos `first_name` y `paternal_surname`.
- **Corrección en Desactivación**: Se eliminó el campo `cancelled_at` en el endpoint `/api/user/deactivate` ya que no existe en el esquema de la base de datos, evitando errores al cancelar membresías.

### 🧩 Widgets (Webflow)
- **User Profile Widget**:
  - Se validó el mapeo de campos para mostrar correctamente Nombre, Teléfono, Fecha de Nacimiento y Dirección consolidada.
  - Se corrigió el fallback del nombre que mostraba el prefijo del correo cuando la API de perfil fallaba debido a los errores de SQL mencionados.
  - Mejorada la visualización de la fecha de próximo pago integrando datos directos de Stripe.

### 🖼️ Gestión de Assets
- **Hosting de Identidad Visual**: Se migró el logo oficial de la CDN externa de Webflow al hosting propio en Vercel para evitar bloqueos de visualización en clientes de correo.
  - Nuevo path: `/public/Identidad/logo-pata-amiga-azul.png`
  - URL pública: `https://app.pataamiga.mx/Identidad/logo-pata-amiga-azul.png`
- **Actualización de Plantillas**: Se actualizaron `email-builder.ts` y `comm.actions.ts` para utilizar el nuevo recurso local, garantizando que el logo sea visible en todos los correos enviados por el sistema.

### 🕒 Automatización de Seguimiento (Cron)
- **Optimización de Cron Job**: Se actualizó el endpoint `/api/cron/missing-info-followup` para procesar de forma robusta e independiente a las Mascotas 1, 2 y 3.
  - Se eliminó la dependencia de tiempos de registro entre mascotas; cada una se evalúa por sus propios días transcurridos.
  - Implementación de **Manual Trigger**: Soporte para parámetros de búsqueda (`memberId`, `petIndex`, `day`, `force`) para disparar seguimientos específicos o generales.
  - Robustez en templates: Mapeo automático de días fuera de rango al template más cercano para evitar errores visuales.
  - Logging avanzado por mascota para facilitar el debugging administrativo.

### 🐾 Refinamiento de Pet Cards Widget (Galería y Carencia)
- **Mejora de Diseño (Modal de Detalles)**:
  - Se optimizó la distribución del modal, reduciendo el ancho de la galería de fotos (**35%**) para priorizar la visualización de datos técnicos (**65%**).
  - Se redujeron las dimensiones de los slots de fotos secundarias para un diseño más compacto y estilizado.
- **Visualización de Carencia**:
  - Implementación de la **barra de progreso de carencia** para mascotas aprobadas.
  - El sistema calcula automáticamente los días restantes basándose en el tipo de mascota y estatus de adopción (120, 150 o 180 días).
  - Integración visual con el sistema de diseño Neo-Brutalista, mostrando porcentaje y fecha objetivo.
- **Optimización Responsiva (Móvil)**:
  - Rediseño completo del modal para pantallas pequeñas, ajustando la galería a un formato panorámico más compacto (`aspect-ratio: 16/9`) para priorizar el contenido.
  - Reorganización de fotos secundarias en una sola fila horizontal con espaciado mínimo.
  - **Corrección de Scroll**: Migración del control de desplazamiento al contenedor principal (`pata-modal-box`) para asegurar que todo el contenido sea accesible.
  - **Ajuste de Altura de Seguridad**: Se redujo la altura máxima (`max-height: 88vh`) para asegurar que el modal no sea cortado por bordes físicos del celular o barras del navegador.

### 🏠 Optimización del Widget Unificado (Dashboard Móvil)
- **Corrección de Layout (Stacking)**: Se corrigió el error de CSS en el dashboard de mascotas aprobadas donde el contenedor flex intentaba usar propiedades de grid, impidiendo el apilamiento vertical en móviles.
- **Tipografía Fluida**: Implementación de `clamp()` en títulos grandes (como "tu periodo de espera") para que se escalen correctamente en celulares sin desbordar el contenedor.
- **Refinamiento de Tarjetas**:
  - Las tarjetas de foto y de información ahora se apilan verticalmente y ocupan el **100%** del ancho disponible en móviles.
  - Se optimizó el espaciado y tamaño de la foto de la mascota para una mejor jerarquía visual en pantallas pequeñas.

### 👤 Impacto funcional
- Los usuarios ahora tienen una mejor legibilidad de los datos de sus mascotas en pantallas grandes.
- Se proporciona visibilidad clara y en tiempo real sobre cuánto tiempo falta para que la cobertura de cada mascota sea efectiva.
- **Mejor experiencia móvil**: El dashboard principal y el modal de detalles ahora son completamente navegables y visualmente armónicos en cualquier dispositivo móvil, eliminando solapamientos de texto y recortes de UI.
- Los administradores pueden ejecutar re-envíos manuales de correos de documentación faltante sin esperar al ciclo diario del cron.
