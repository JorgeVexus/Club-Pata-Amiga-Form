# Changelog 2026-06-30

## ✨ Features

### Integración Completa Lynsales (LeadConnector)

Se completó la integración con Lynsales (LeadConnector) para gestionar de forma automatizada los 7 campos personalizados y los 6 eventos clave del flujo de membresía y mascotas en el CRM.

#### Cambios implementados:

1. **Servicio CRM Centralizado (`src/services/crm.service.ts`)**
   - Mapeo centralizado de los 7 campos personalizados (IDs de custom fields de la agencia).
   - Implementación de la etiqueta `'miembro activo'` al completarse el registro y pago.
   - Creación del método `syncMembership` para sincronizar campos en formato `{ id, value }` con el CRM de forma no bloqueante.
   - Creación del método `removeContactTags` para remover la etiqueta de miembro activo en caso de cancelación del usuario.

2. **Extracción y Mapeo de Stripe (`src/lib/stripe-membership.ts` - nuevo)**
   - Extracción de datos reales de pago y renovación desde la suscripción de Stripe (`getStripeMembershipFields`).
   - Mapeo legible de métodos de pago (`card` -> `Tarjeta`, `oxxo` -> `OXXO`, etc.).
   - Utilidad `toCrmDate` para formateo estandarizado de fechas en formato `YYYY-MM-DD`.

3. **Eventos Conectados en Rutas de API:**
   - **Registro y Pago Exitoso:** Vinculación en `api/admin/members/[id]/approve/route.ts` para sincronizar estatus activo, tag, costo, método de pago y renovación.
   - **Aprobación de Mascotas:** Sincronización al aprobar el estatus de las mascotas en `api/admin/members/[id]/pets/[petId]/status/route.ts`.
   - **Resincronización Manual:** Creación de endpoint de administración `api/admin/members/[id]/sync-crm/route.ts`.
   - **Renovación y Fallos en Webhook:** Captura de eventos `invoice.payment_succeeded` y `invoice.payment_failed` en el webhook de Stripe para actualizar fechas de renovación y estatus de cobro.
   - **Cancelación Voluntaria:** Actualización a estatus `cancelado` y eliminación de la etiqueta en el flujo de desactivación (`api/user/deactivate/route.ts`).

## 🐛 Bug Fixes

### Corrección de Extracción de Suscripción en Stripe Webhook (`src/app/api/stripe/webhook/route.ts`)

- Se corrigió un problema donde el webhook de Stripe fallaba al extraer el ID de suscripción debido a diferencias entre las versiones de la API de Stripe.
- Se implementó una lógica de fallback robusta para buscar el ID de suscripción en:
  1. `invoice.subscription`
  2. `invoice.parent?.subscription_details?.subscription`
  3. `invoice.lines?.data?.[0]?.subscription`
  4. `invoice.lines?.data?.[0]?.parent?.subscription_item_details?.subscription`
