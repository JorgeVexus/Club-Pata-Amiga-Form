# Changelog - 2026-05-12

## Resumen del día
Hoy se realizaron optimizaciones visuales y funcionales en el **Widget de Membresía Unificado**, se corrigieron los dominios de los correos de embajadores para asegurar la correcta redirección a la aplicación, y se reforzó la identidad visual premium del Club Pata Amiga. También se inició la planeación de la arquitectura de usuarios multi-rol.

## Cambios Realizados

### Widgets del Miembro (Frontend)
- **style (Widget Unificado)**: Optimización profunda de `unified-membership-widget.js`.
  - **Banner Premium**: Se rediseñó el banner de "Certificado Pendiente" (opcional para mascotas senior 10+ años) con una estética Neo-Brutalista:
    - Colores turquesa de marca con bordes negros de 3px y sombras sólidas.
    - Tipografía premium (`Fraiche` y `Outfit`).
    - Botón de acción naranja (`#FE8F15`) con efectos hover mejorados.
  - **Control de Modal**: Se redimensionó el modal de actualización (`max-width: 600px`, `max-height: 85vh`) para garantizar que siempre sea interactuable en cualquier resolución de pantalla, añadiendo scroll interno.
  - **Previews de Archivos**: Se implementó la clase CSS `.pata-upload-preview` para restringir el tamaño de las fotos y certificados subidos.
    - Uso de `object-fit: cover` para mantener la proporción sin deformar la interfaz.
    - Inclusión de un overlay de nombre de archivo (`.pata-upload-filename`) para mejorar el feedback de subida.
  - **Higiene Visual**: Se ajustaron los estilos de alerta internos del modal para mantener la coherencia con el nuevo sistema de diseño.
  - **Gestión de Archivos**: Se añadió un botón para remover archivos seleccionados antes de subir y lógica para re-vincular inputs dinámicamente.
  - **Bloqueo de Scroll**: Implementación de la clase `.pata-no-scroll` para bloquear el scroll del fondo mientras el modal está abierto, mejorando la UX en móviles.
  - **Constraints de Preview**: Se forzó el tamaño de los previews a 120px con bordes neo-brutalistas para evitar desbordamientos del modal.
- **feat (Widget Unificado)**: Refactor completo de alertas de documentación.
  - Consolidación de `renderOptionalDocsBanner` en `renderAlertBanners` para mostrar alertas dinámicas (fotos + certificados) en una sola vista con iconos de Material Symbols.
  - Sincronización de lógica `isSenior` y soporte para campos `photo_url` / `primary_photo_url`.
- **fix (Widget de Embajadores)**: Corrección crítica en la detección de rol en `ambassador-widget.js`.
  - Migración de detección por email (endpoint de administración restringido) a detección por `memberstackId` (endpoint público autorizado).
  - Implementación de logs de consola detallados con el prefijo `[AmbassadorWidget]` para monitoreo en tiempo real.
  - Normalización de la lógica para manejar respuestas tanto de búsqueda (array) como de ID único (objeto).
- **feat (Widget de Tarjetas)**: Ajustes en `pet-cards-widget.js`.
  - Cambio de título a "Activación de los beneficios".
  - Ocultación automática del campo si la mascota no ha sido aprobada.
  - Sincronización de cálculos de carencia con `carencia.utils.ts`.

### Planeación y Arquitectura
- **docs**: Creación del documento de planeación para el **Sistema de Usuarios Multi-Rol** (`Documentacion/planeacion/2026-05-12-sistema-multi-roles.md`).
  - Definición de la estructura para permitir que un mismo usuario tenga roles de Miembro, Embajador y Centro de Bienestar simultáneamente.
- **docs**: Documentación del plan de corrección para el widget de embajadores (`Documentacion/planeacion/2026-05-13-fix-ambassador-detection.md`).

### Backend (API Routes)
- **feat (API de Embajadores)**: Optimización del endpoint `GET /api/ambassadors/by-memberstack`.
  - Ahora devuelve el objeto detallado del embajador, incluyendo conteo de referidos y lista de referidos recientes en una sola llamada.
  - Mejora en la consistencia de la estructura de datos (`data`) para facilitar el consumo desde widgets externos.
### Embajadores y Comunicaciones
- **Corrección de Redirecciones**: Se actualizaron los botones "Ir a mi dashboard" en los flujos de selección y cambio de código para que apunten al dashboard externo oficial: `https://www.pataamiga.mx/embajadores/dashboard`.
- **Actualización de Dominios**: Se reemplazaron las referencias a `clubpataamiga.com` por `www.pataamiga.mx` en los previews de enlaces de referido y en el pie de página de los correos electrónicos.
- **Consistencia de Notificaciones**: Se actualizaron las acciones de comunicación (`ambassador-comm.actions.ts`) para que todos los botones de acción en los correos (comisiones, códigos establecidos) dirijan al nuevo dashboard.

### Tareas Técnicas Realizadas
- Modificación de `src/app/embajadores/seleccionar-codigo/page.tsx` para usar `window.location.href`.
- Actualización de `src/app/embajadores/cambiar-codigo/page.tsx`.
- Refactorización de componentes `ReferralCodeSelector` y `ReferralCodeChanger` para mostrar el dominio correcto.
- Sincronización de enlaces en `src/app/actions/ambassador-comm.actions.ts`.

## Impacto Funcional
- Los embajadores ahora pueden ver correctamente su panel de control y estado de solicitud sin que el sistema les pida registrarse nuevamente.
- Se eliminaron los errores 401 (Unauthorized) al cargar el widget de embajadores para usuarios normales.
- Los usuarios con mascotas senior ahora ven una invitación mucho más atractiva y clara para reducir su periodo de carencia.
- Se eliminaron los problemas de UX donde fotos muy grandes bloqueaban el acceso a los botones de cierre del modal.
- Los correos de aprobación de embajadores ahora dirigen correctamente a la aplicación (`app.pataamiga.mx`), eliminando confusiones de navegación.
- Preparada la base para la evolución hacia un sistema de roles más flexible.
