# Changelog - 2026-05-10

## Resumen del día
Hoy se corrigió un fallo crítico en la lógica de carencia (tiempo de espera) y se refinó el sistema de solicitudes del Fondo Solidario para alinearlo con los requerimientos operativos y de diseño de Club Pata Amiga.

## Cambios Realizados

### Backend & API
- **fix**: Se corrigió el cálculo de carencia para que inicie al momento de la **aprobación por el admin** en lugar de al momento de la creación del registro.
  - El endpoint `POST /api/admin/members/[id]/pets/[petId]/status` ahora puebla automáticamente `waiting_period_start` y `waiting_period_end` tras la aprobación.
  - Se añadieron campos adicionales (`is_adopted`, `is_mixed_breed`, etc.) en la selección previa a la actualización para garantizar cálculos precisos de días (120/150/180).

### Widgets (Frontend)
- **refactor**: Se actualizaron los widgets `pet-cards-widget.js`, `solidarity-dashboard.js` y `unified-membership-widget.js`.
  - Ahora muestran la fecha de fin de carencia con el formato `D de M de YYYY`.
  - Se añadió lógica reactiva para mostrar el botón "Pedir Apoyo" solo cuando la carencia ha finalizado.
- **feat**: Implementación de sincronización en tiempo real para el chat de solidaridad usando Supabase Realtime.
- **fix**: Mejoras en la visualización de adjuntos multimedia (video/imagen/pdf) y descarga directa.

### Sistema de Notificaciones en Tiempo Real y Deep Linking
- **feat (Push Notifications)**: Implementación de notificaciones automáticas en el chat de solidaridad.
  - El sistema ahora genera notificaciones en tiempo real cada vez que se recibe un nuevo mensaje.
  - **Administradores**: Reciben notificaciones cuando un miembro envía un mensaje o adjunto.
  - **Miembros**: Reciben notificaciones cuando el soporte responde a su solicitud.
- **feat (Deep Linking)**: Las notificaciones actúan como enlaces directos ("Deep Links").
  - Al hacer clic en una notificación, el administrador es llevado directamente al chat específico dentro del dashboard, abriendo el modal de detalle automáticamente.
  - Los miembros son redirigidos directamente a la página de detalle de su solicitud en el portal de Webflow.
- **fix (UX)**: Personalización de mensajes de notificación incluyendo el nombre de la mascota involucrada para mayor contexto.
- **Impacto**: Comunicación instantánea y sin fricciones, asegurando que ninguna respuesta quede sin ser atendida por ambas partes.

### Correcciones de Notificaciones (Post-Implementación)
- **fix (Messaging)**: Corregido error de base de datos que impedía la entrega de notificaciones del chat.
  - El campo `type` en la tabla `notifications` tiene una restricción CHECK que no permitía el valor `'solidarity'`.
  - Se cambió el tipo a `'account'` (permitido) para asegurar la inserción y entrega correcta.
- **feat (Logging)**: Añadidos logs de servidor para monitorear el flujo de notificaciones en tiempo real.
- **fix (UI Admin)**: Corregida la visibilidad y el desplazamiento (scroll) del chat en el modal de detalle de solidaridad de administración.
  - Se añadió `min-height: 0` a los contenedores de flexbox para asegurar que el historial de chat no empuje el área de entrada fuera de la pantalla.
  - El área de entrada de mensajes ahora permanece fija en la parte inferior del modal.
- **feat (Admin Attachments)**: Añadida la capacidad para que los administradores adjunten archivos (imágenes, PDFs, videos) directamente desde el chat del dashboard.
  - Nuevo endpoint API `/api/upload/solidarity-attachment` para gestionar los adjuntos del chat de forma segura.
  - Sincronización completa con la vista del usuario: ambos pueden ahora intercambiar archivos multimedia.
- **style**: Añadido título de "Chat" al área de conversación para mayor claridad visual y consistencia con la interfaz de usuario.
- **fix (UI)**: Optimizado el layout del campo de mensaje en el modal administrativo. Se aumentó la altura del `textarea` a 80px y se garantizó que ocupe el ancho completo disponible, eliminando la apariencia "comprimida".
- **fix (Notifications)**: Corregida la extracción del ID de Memberstack en el sistema de notificaciones del chat. El sistema ahora maneja correctamente las respuestas de Supabase tanto en formato objeto como array, asegurando que las notificaciones lleguen consistentemente a la campanita del usuario.
- **debug**: Añadido registro (logging) de servidor detallado para rastrear el envío de notificaciones y la identidad de los receptores en el módulo de solidaridad.
- **fix (Messaging API)**: Resuelto error 400 (Bad Request) al enviar mensajes desde el panel administrativo. Se corrigió la discrepancia entre el formato de envío del frontend (snake_case) y la expectativa del backend (camelCase), haciendo que la API sea ahora compatible con ambos formatos para mayor robustez.
- **feat (Real-Time UI)**: Optimización del intervalo de polling en el widget de notificaciones.
  - Se redujo de 12 segundos a **5 segundos** para una experiencia más cercana al tiempo real.
- **feat (Deep Linking & Navigation)**: Mejora del flujo de navegación desde notificaciones.
  - Las notificaciones para miembros ahora redirigen directamente a la página de detalle de la solicitud específica (`/miembros/detalle-solicitud?id=...#chat`).
  - Se implementó un sistema de **auto-scroll** en el widget de detalle que desplaza la página automáticamente hasta la sección de chat cuando se detecta el hash `#chat` en la URL.
- **style (Admin UX)**: Ampliación del área de redacción de mensajes en el detalle administrativo.
  - Se incrementó la altura mínima de la `textarea` de 80px a **120px** y se aseguró el uso del ancho total para una mejor experiencia de escritura.
- **fix (API Routing)**: Actualización de los enlaces de notificación generados en el backend para soportar la nueva lógica de navegación directa al chat.
- **fix (Redirection UX)**: Corregido error que abría el modal de detalles en lugar de redirigir a la página de la solicitud.
  - El backend ahora genera **URLs absolutas** (`https://www.pataamiga.mx/...`) para garantizar la redirección correcta entre subdominios.
  - El widget de notificaciones se actualizó con un sistema de **Smart Mapping robusto** que maneja parámetros de consulta (query params) y fragmentos (#chat) de forma precisa.
