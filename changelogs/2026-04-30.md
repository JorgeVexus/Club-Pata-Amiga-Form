# Changelog - 2026-04-30

## 🛠️ Correcciones y Mejoras Críticas

### 💳 Finanzas y Suscripciones (Stripe)
- **Fix: Cancelación automática de suscripciones al eliminar miembros.**
  - Se identificó un error crítico donde los miembros eliminados del Dashboard Admin (Memberstack + Supabase) seguían teniendo suscripciones activas en Stripe, resultando en cobros indebidos.
  - **Cambios en Backend:**
    - Se actualizó el endpoint `DELETE /api/admin/members/[id]/delete` para incluir un paso de cancelación inmediata en Stripe antes de borrar al usuario de Memberstack.
    - Se actualizó el endpoint `POST /api/admin/members/bulk-delete` con la misma lógica para eliminaciones masivas.
    - **Lógica de Fallback:** Si el `stripe_customer_id` no está en Supabase, el sistema busca al cliente en Stripe por su correo electrónico antes de proceder con la eliminación.
  - **Impacto:** Garantiza que cualquier usuario dado de baja desde el panel de "Baja de Usuarios" deje de recibir cargos recurrentes automáticamente.
- **Mejora: Distribución de Facturación 70/30 y Etiquetas SAT.**
  - Se implementó la visualización de la distribución de costos (70% Empresa / 30% Asociación Civil) en el dashboard de facturación.
  - **Backend:** Se enriqueció el endpoint de facturación integrando la API de Memberstack Admin para obtener los montos reales de los planes de cada miembro.
  - **Frontend:** Se agregaron las columnas "Empresa (70%)" y "Asociación (30%)", se restauró la columna de **C.P.** y se reemplazaron los códigos numéricos por descripciones legibles.
  - **Copy to Clipboard:** Se implementaron botones discretos para copiar datos directamente al portapapeles, optimizando los montos como números planos para facilitar el pegado en **Contalink**.
  - **Impacto:** Mejora la transparencia contable y facilita el proceso de facturación dual para el equipo administrativo.

## 📝 Notas de Administración
- **Acción Manual Requerida:** Para los 3 casos específicos reportados hoy (ya eliminados previamente), se debe realizar la cancelación y reembolso manual directamente en el Dashboard de Stripe, ya que sus registros ya no existen en el sistema local para procesar la automatización.
