# Bitácora de Cambios - Sesión 11/Abril/2026

En esta sesión se optimizó la usabilidad del componente de selección de fecha para dispositivos móviles y se implementaron reglas de negocio críticas para el registro de mascotas cachorras.

---

## 1. Entrada Manual de Fecha en Móvil (UX/UI)

Se rediseñó la interacción del componente `DatePicker` para permitir que los usuarios introduzcan su fecha de nacimiento manualmente mediante el teclado numérico en dispositivos móviles, sin perder la funcionalidad del calendario nativo.

### 🛠️ Detalles Técnicos
- **Optimización de Clip-Path**: Se modificó el área de activación del input nativo (`type="date"`) en `DatePicker.module.css`. Ahora, el input nativo solo cubre el **30% derecho** del campo (área del icono).
- **Liberación del Input de Texto**: El **70% izquierdo** del componente ahora permite el foco directo sobre el `input` de texto subyacente, disparando el teclado numérico (`inputMode="numeric"`) en iOS y Android.
- **Seguridad de Pantalla**: Al utilizar el comportamiento nativo del navegador para el trigger del calendario, se garantiza que el selector de fecha se mantenga siempre dentro de los límites visibles de la pantalla, resolviendo errores de desbordamiento previos.

---

## 2. Validación de Edad Mínima para Mascotas (Paso 2)

Se implementó una nueva regla de negocio en el flujo de registro de mascotas (`Step2PetBasic.tsx`) para asegurar que solo se registren mascotas con una edad mínima de **4 meses**.

### 🛠️ Detalles Técnicos
- **Lógica de Validación**: Se actualizó la función `validateForm` para calcular el total de meses independientemente de la unidad seleccionada (años/meses).
- **Alerta Visual Dinámica**: Implementación de un nuevo `infoBox` con estilo de **error** (rojo) que se muestra instantáneamente cuando la edad ingresada es menor a 4 meses.
- **Bloqueo de Navegación**: El formulario ahora impide avanzar al siguiente paso ("Continuar") si no se cumple el requisito de edad mínima, mostrando un mensaje de error específico en el campo de edad.

---

## 3. Validación de RUAC para Mascotas (Paso 5)

Se actualizó la validación del campo RUAC en el componente `Step5CompletePet.tsx` para ajustarlo al formato oficial de 11 caracteres alfanuméricos.

### 🛠️ Detalles Técnicos
- **Validación de formato**: Se cambió la expresión regular de `/^[A-Z0-9]{10}$/` a `/^[A-Z0-9]{11}$/` para aceptar exactamente 11 caracteres
- **Longitud del input**: Se aumentó `maxLength` de 10 a 11 caracteres
- **Placeholder actualizado**: Se cambió de "Ej: A1B2C3D4E5" a "Ej: A1B2C3D4E5F"
- **Indicador de validación**: El ícono ✨ ahora se muestra únicamente cuando se ingresan correctamente los 11 caracteres del RUAC
- **Mensaje de error**: Actualizado para indicar que se requieren exactamente 11 caracteres alfanuméricos

---

## 4. Mapeo de Edad en Registro de Mascotas (Widget)

Se corrigió un error en el widget de tarjetas de mascotas donde las nuevas mascotas registradas aparecían con edad "No especificada".

### 🛠️ Detalles Técnicos
- **Sincronización de API**: Se actualizó el endpoint `POST /api/user/pets/add` para mapear los campos `age_value` y `age_unit` desde el payload del widget hacia la tabla `pets` en Supabase.
- **Campos Adicionales**: Se incluyó el mapeo de colores (`coat_color`, `nose_color`, `eye_color`) para asegurar consistencia en el perfil completo de la mascota desde el momento de su creación.

---

## 5. Historias de Adopción para Mascotas Mestizas (Widget)

Se implementó una nueva funcionalidad interactiva que permite a los usuarios añadir la historia de adopción a sus mascotas mestizas directamente desde el panel de su manada.

### 🛠️ Detalles Técnicos
- **UI de Historia en Modal**: Se integró un componente de captura de texto en el modal de detalles (`showDetails`) del widget. Este componente solo aparece si la mascota es mestiza y carece de historia.
- **Diseño Premium**: El bloque utiliza gradientes suaves y sombras modernas para integrarse con la estética del proyecto.
- **Actualización de API**: Se modificó el endpoint `POST /api/user/pets/[petId]/update` para permitir la recepción del campo `adoptionStory` y persistirlo en la base de datos, incluso si la mascota está en estados que normalmente bloquean ediciones masivas (como `pending`).
- **Callback de Refresco**: El widget recarga automáticamente la información tras guardar la historia para ofrecer feedback inmediato al usuario.

---

## 6. Redirección de Recuperación de Contraseña

Se ajustó el flujo de recuperación de contraseña para garantizar que el usuario sea redirigido correctamente después de completar el proceso.

### 🛠️ Detalles Técnicos
- **Ajuste de Redirect**: Se configuró el parámetro de redirección en las llamadas de autenticación para que el destino post-cambio de clave sea el Dashboard de Usuario (`/dashboard`) en lugar del login o el inicio.
- **Validación de Sesión**: Se mejoró la lógica de persistencia para asegurar que la sesión de Memberstack se reconozca inmediatamente tras la redirección, evitando la necesidad de un login manual adicional.
