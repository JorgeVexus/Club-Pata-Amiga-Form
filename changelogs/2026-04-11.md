# Bitácora de Cambios - Sesión 11/Abril/2026

En esta sesión se optimizó la usabilidad del componente de selección de fecha para dispositivos móviles y se implementaron reglas de negocio críticas para el registro de mascotas cachorras.

---

## 1. Entrada Manual de Fecha en Móvil (UX/UI)

Se rediseñó la interacción del componente `DatePicker` para permitir que los usuarios introduzcan su fecha de nacimiento manualmente mediante el teclado numérico en dispositivos móviles, sin perder la funcionalidad del calendario nativo.

### 🛠️ Detalles Técnicos
- **Optimización de Clip-Path**: Se modificó el área de activación del input nativo (`type="date"`) en `DatePicker.module.css`. Ahora, el input nativo solo cubre el **30% derecho** del campo (área del icono).
- **Liberación del Input de Texto**: El **70% izquierdo** del componente ahora permite el foco directo sobre el `input` de texto subyacente, disparando el teclado numérico (`inputMode="numeric"`) en iOS y Android.
- **Seguridad de Pantalla**: Al utilizar el comportamiento nativo del navegador para el trigger del calendario, se garantiza que el selector de fecha se mantenga siempre dentro de los límites visibles de la pantalla, resolviendo errores de desbordamiento previos.

---

## 2. Validación de Edad Mínima para Mascotas (Paso 2)

Se implementó una nueva regla de negocio en el flujo de registro de mascotas (`Step2PetBasic.tsx`) para asegurar que solo se registren mascotas con una edad mínima de **4 meses**.

### 🛠️ Detalles Técnicos
- **Lógica de Validación**: Se actualizó la función `validateForm` para calcular el total de meses independientemente de la unidad seleccionada (años/meses).
- **Alerta Visual Dinámica**: Implementación de un nuevo `infoBox` con estilo de **error** (rojo) que se muestra instantáneamente cuando la edad ingresada es menor a 4 meses.
- **Bloqueo de Navegación**: El formulario ahora impide avanzar al siguiente paso ("Continuar") si no se cumple el requisito de edad mínima, mostrando un mensaje de error específico en el campo de edad.
