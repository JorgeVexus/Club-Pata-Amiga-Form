# Changelog - 2026-05-19

## Resumen de Cambios
Hoy se implementó el widget de visibilidad condicional del botón de acceso al Fondo Solidario en Webflow (`#fondo`) basado en la sesión de Memberstack y el estado/tiempo de espera (carencia) de las mascotas del usuario.

## Cambios Realizados

### Frontend (Widgets de Webflow)
- **Nuevo Widget de Visibilidad (`public/widgets/solidarity-button-visibility.js`):** Script ligero y autónomo (IIFE) que espera a que Memberstack cargue, consulta las mascotas del usuario a través de la API `/api/user/pets` y aplica las reglas de tiempo de espera (carencia). Muestra el botón con ID `#fondo` si el miembro tiene al menos una mascota aprobada (`approved`), activa y sin días restantes de carencia.
- **Logs de Depuración Robustos:** Se agregaron console logs descriptivos a lo largo de todo el flujo para facilitar el rastreo y diagnóstico directo en Webflow.
- **Actualización de Estilos del Widget (`public/widgets/solidarity-dashboard.js`):**
  - Ajustado `.pata-dashboard-wrapper` para cambiar `max-width` a `100%` y agregar `align-items: center`.
  - Añadido ancho `width: 100%` a `header` y `main` (hijos de primer nivel del wrapper flex) para evitar que colapsen.
  - Reestructurado `.pata-main-card` con `max-width: 1400px`, `width: calc(100% - 40px)` y `margin: 20px auto` para asegurar un centrado impecable en pantallas grandes.
  - Añadido color de fondo `#eaeaea` a `.pata-content-layout`.

### Backend (Middleware de CORS)
- **Soporte de CORS para Webflow (`src/middleware.ts`):** Se añadió de forma dinámica soporte de orígenes autorizados para subdominios `.webflow.io` en el middleware de Next.js, previniendo fallos por bloqueo CORS durante pruebas en Webflow Staging.

## Impacto Funcional
- **Control Dinámico del Menú:** Los miembros sólo verán el botón de acceso al Fondo Solidario si realmente tienen cobertura activa de acuerdo a las reglas de carencia (180, 150, 120 o 90 días según adopción, raza mixta o uso de código de referido/embajador).
- **Estabilidad de Testing:** Facilita las pruebas de desarrollo sobre los subdominios de staging de Webflow sin bloqueos de origen cruzado.
- **Mejora Visual en Webflow:** Mayor adaptabilidad en pantallas ultra-anchas para el dashboard de manada, asegurando que la tarjeta principal de estado del fondo esté centrada y que el layout de contenido resalte correctamente con su fondo `#eaeaea`.
