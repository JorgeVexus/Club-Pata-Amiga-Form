# Changelog - 2026-05-19

## Resumen de Cambios
Hoy se implementó el widget de visibilidad condicional del botón de acceso al Fondo Solidario en Webflow (`#fondo`) basado en la sesión de Memberstack y el estado/tiempo de espera (carencia) de las mascotas del usuario.

## Cambios Realizados

### Frontend (Widgets de Webflow)
- **Nuevo Widget de Visibilidad (`public/widgets/solidarity-button-visibility.js`):** Script ligero y autónomo (IIFE) que espera a que Memberstack cargue, consulta las mascotas del usuario a través de la API `/api/user/pets` y aplica las reglas de tiempo de espera (carencia). Muestra el botón con ID `#fondo` si el miembro tiene al menos una mascota aprobada (`approved`), activa y sin días restantes de carencia.

## Impacto Funcional
- **Control Dinámico del Menú:** Los miembros sólo verán el botón de acceso al Fondo Solidario si realmente tienen cobertura activa de acuerdo a las reglas de carencia (180, 150, 120 o 90 días según adopción, raza mixta o uso de código de referido/embajador).
