# Changelog - 2026-05-13

## Resumen del día
Hoy se resolvió el bloqueo crítico de navegación en el **Dashboard de Embajadores**, donde los embajadores aprobados quedaban atrapados en un bucle solicitando la selección de un código de referido ya existente. Se optimizaron los endpoints de la API y la lógica del widget frontend para garantizar una detección de estado robusta y precisa.

## Cambios Realizados

### Backend (API Routes)
- **fix (API de Embajadores)**: Actualización del endpoint `GET /api/ambassadors/by-memberstack`.
  - Se incluyeron los campos `referral_code_status` y `referral_code_selected_at` en la respuesta JSON.
  - Se añadió el campo `referrals_count` (alias de `total_referrals`) para asegurar la compatibilidad total con el widget de Webflow.
  - Implementación de un **Fallback de Seguridad**: Ahora el sistema marca automáticamente como `active` cualquier código de referido que no sea de tipo temporal (`TMP`), protegiendo la experiencia del usuario contra posibles desincronizaciones de estado en la base de datos.
- **fix (Generación de Links)**: Ajuste en `/api/ambassadors/[id]/generate-code-link` para asegurar que el link de selección generado use siempre el dominio de aplicación correcto (`app.pataamiga.mx`).

### Widgets del Miembro (Frontend)
- **fix (Widget de Embajadores)**: Optimización de la lógica de detección en `ambassador-widget.js`.
  - Se flexibilizó la validación de `hasActiveCode`. Ahora el widget permite el acceso al dashboard si el estado es `active` O si el código detectado no es un código temporal.
  - Se implementó un sistema de **Logging Detallado** (`[AmbassadorWidget] Referral code check`) para facilitar el monitoreo proactivo de estados de embajador desde la consola del navegador.
  - Mejora en la robustez del mapeo de datos recibidos de la API.

### Navegación y UX
- **fix (Selección de Código)**: Corrección de la URL de redirección final en el componente `SeleccionarCodigoPage` (`src/app/embajadores/seleccionar-codigo/page.tsx`).
  - El botón de éxito ahora redirige directamente al dashboard oficial en producción: `https://www.pataamiga.mx/embajadores/dashboard`.
- **fix (Comunicaciones)**: Sincronización de enlaces en los correos electrónicos de aprobación para asegurar que apunten al dominio de la aplicación (`app.pataamiga.mx`).

## Tareas Técnicas Realizadas
- Modificación de la consulta SQL en `src/app/api/ambassadors/by-memberstack/route.ts`.
- Refactorización de la función `renderApproved` en `public/widgets/ambassador-widget.js`.
- Verificación de consistencia en `src/app/api/ambassadors/referral-code/save/route.ts`.
- Validación del build de producción con `npm run build` (Exitoso).

## Impacto Funcional
- **Resolución de Bucle**: Los embajadores aprobados que ya han elegido su código ahora pueden acceder directamente a su dashboard sin ser redirigidos erróneamente a la pantalla de selección.
- **Sincronización Total**: Se eliminaron las discrepancias entre el estado real del embajador en la base de datos y lo que el widget frontend interpretaba.
- **Mejora en Soporte**: Los nuevos logs en el widget permiten identificar rápidamente si un usuario tiene problemas con su código de referido.
- **Consistencia de Marca**: Todas las redirecciones críticas ahora apuntan consistentemente a los dominios de producción configurados.
