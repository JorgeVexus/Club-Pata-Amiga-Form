# Bitácora de Cambios - Sesión 09/Abril/2026

En esta sesión se implementó una funcionalidad esencial de seguridad y usabilidad para el formulario de inicio de sesión en Webflow, igualando la experiencia premium del flujo de registro principal.

---

## 1. Widget de Visibilidad de Contraseña (Webflow)

Se desarrolló un widget ligero en JavaScript vainilla diseñado específicamente para integrarse con Webflow y Memberstack, permitiendo a los usuarios ver u ocultar su contraseña mediante un icono de "ojo".

### 🛠️ Detalles Técnicos
- **Implementación Universal**: El script (`password-toggle.js`) detecta automáticamente campos de contraseña mediante el atributo `data-ms-member="password"` o el tipo nativo `password`.
- **Integración con Webflow**: Uso de atributos personalizados (`data-ms-toggle="password"`) para facilitar la configuración en el diseñador de Webflow sin necesidad de modificar el código del widget.
- **Micro-interacciones**:
  - Soporte para alternancia de iconos (show/hide) mediante data-attributes.
  - Feedback visual inmediato al cambiar la opacidad y los estados ARIA del botón.

### 🐛 Corrección de Interferencia (Problema de los 2 Clics)
Se identificó y resolvió un problema donde la funcionalidad requería dos clics para activarse/desactivarse debido a la interferencia con los listeners nativos de Webflow y Memberstack.
- **Fase de Captura (Capture Phase)**: Se migró el listener a la fase de captura del DOM para asegurar que el widget procese la acción antes que cualquier otra lógica externa.
- **Aislamiento de Eventos**: Implementación de `stopImmediatePropagation()` para garantizar que el cambio de tipo de input no sea revertido por scripts de terceros en el mismo ciclo de evento.
- **Protección de Inicialización**: Candado interno para evitar ejecuciones duplicadas en entornos de carga dinámica.

## 2. Reparación de Restablecimiento de Contraseña (Memberstack)

Se resolvieron errores fatales que impedían a los usuarios restablecer su contraseña desde el modal nativo de Memberstack en Webflow.

### 🛠️ Detalles Técnicos
- **Programación Defensiva**: Se actualizó `script.js` con verificaciones de existencia para evitar errores de `null.value` cuando el script global se ejecuta en páginas de login o modales.
- **Interoperabilidad de Eventos**: Se modificó el widget de `password-toggle.js` eliminando `stopImmediatePropagation()`. Esto permite que Memberstack reciba los eventos necesarios para procesar sus formularios internos en Svelte.
- **Parche de Seguridad para Webflow**: Creación de un script inyectable que detecta el modal de Memberstack a través de `MutationObserver` y aplica capas de protección para evitar interferencias de scripts globales.

## 3. Configuración de Email y Dominios (Entregabilidad)

Se solucionó un conflicto de dominios entre Memberstack y la aplicación Next.js, mejorando la entregabilidad de correos y eliminando el etiquetado como SPAM.

### 🛠️ Detalles Técnicos
- **Migración a Subdominio**: Se migró el motor de notificaciones de la aplicación Next.js al subdominio `app.pataamiga.mx` en Resend.
- **Resolución de Conflictos**: Esta migración libera el dominio raíz (`pataamiga.mx`) para su uso exclusivo en la autenticación de Memberstack, resolviendo el error de duplicidad en Resend.
- **Sincronización de Configuración**: Actualización del App ID de Memberstack en `AGENTS.md` y variables de entorno para asegurar que el desarrollo coincida con la aplicación "Live".
