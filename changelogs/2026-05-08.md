## [Solidarity Fund Request] Mobile Optimization & Data Persistence

### Added
- **Mobile Solidarity Widget**: Fully responsive, multi-step support request flow with scroll-lock, anti-overflow measures, and app-like transitions.
- **Data Persistence Architecture**: Created `allied_centers` table and extended `solidarity_requests` to support 100% of the form fields (including Medical Emergencies).
- **Allied Centers API**: New endpoint `/api/solidarity/allied-centers` to dynamically populate the provider list.
- **Multi-Document Support**: Implemented structured metadata storage for evidence, prescriptions, and receipts in Supabase.

### Fixed
- **Pet Photo Rendering**: Updated logic to use `photo_url` with a robust fallback to `primary_photo_url` and a local placeholder, ensuring images load correctly in the pet grid.
- **Enforced Access Eligibility**: Restricted access to the solidarity fund to only pets with `approved` status and an expired waiting period, preventing premature requests.
- **Form Data Sync**: Resolved mapping issues between UI IDs and state keys (e.g., Description, Amounts) that were preventing the submit button from activating.
- **Mobile UX Refinement**: Removed redundant paddings in `.pata-reveal` and adjusted `.pata-form-page` spacing to 10px to eliminate horizontal overflow on small devices.
- **Data Validation**: Implemented strict numeric input for Postal Codes (5-digit limit) and Amounts to ensure data integrity.

### Added
- **Missing Receipt Field**: Added a dedicated file upload for receipts/invoices, which was previously a required but hidden field for reimbursements.
