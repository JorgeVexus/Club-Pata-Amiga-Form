# Changelog - 2026-07-04

## feat: add wellness center branches

**Commit:** `db742c3`

Se implemento soporte para multiples sucursales en Centros de Bienestar.

### Cambios principales
- Se creo la tabla `wellness_center_locations` para permitir sucursales ilimitadas por centro.
- Se agregaron politicas RLS para que cada centro pueda gestionar sus propias ubicaciones y el `service_role` mantenga acceso administrativo.
- El widget `wellness-center-widget` ahora pregunta si el negocio tiene mas de una sucursal y permite agregar/eliminar sucursales adicionales.
- Las sucursales se guardan desde `/api/wellness/update` y se devuelven desde `/api/wellness/me`.
- El endpoint `/api/wellness/locations` ahora puede exponer puntos por sucursal para el mapa publico, manteniendo fallback a la ubicacion principal.
- El dashboard admin muestra las sucursales registradas dentro del detalle del Centro de Bienestar.
- Se agrego cobertura de prueba para el contrato de sucursales wellness.

### Impacto funcional
- Los aliados pueden registrar 2, 10, 20, 30 o mas sucursales sin cambios futuros en columnas.
- Los servicios y beneficios siguen viviendo en el centro principal, como se definio para esta etapa.
- Admin puede revisar las ubicaciones capturadas antes de aprobar o dar seguimiento al centro.
