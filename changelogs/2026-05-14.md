# Changelog - 2026-05-14

## Corrección de Sincronización con LynSales CRM

### Fixed
- **Sincronización de Membresía**: Se resolvió el problema donde el tipo de membresía (Mensual/Anual) y el costo no se enviaban correctamente al CRM al aprobar un miembro.
- **Detección de Planes**: Se corrigió el mapeo de IDs de planes de Memberstack en la ruta de aprobación (`/api/admin/members/[id]/approve`).
- **Errores de Tipado**: Se corrigieron errores de TypeScript en `src/services/crm.service.ts` relacionados con la interfaz `UpdateResponse`.

### Added
- **Logging Detallado**: Se implementó un sistema de logs en el backend para rastrear el payload exacto enviado a LynSales y la respuesta recibida del servidor.
- **Sistema de Fallback**: Se añadió un flujo de recuperación de datos que utiliza información de **Supabase** como respaldo.
- **Sincronización Impulsada por Frontend**: Se implementó la transmisión directa de tipo de membresía y costo desde el Dashboard Administrativo a la API de aprobación para garantizar que los datos en el CRM coincidan con lo que ve el administrador.
- **Documentación de Planeación**: Se creó el documento de planeación técnica para la sincronización robusta.

### Changed
- **Dashboard Administrativo**: Se actualizó el modal de detalles de miembro para calcular y enviar metadata de membresía en el flujo de aprobación.
- **Servicio CRM**: Se optimizó la función `updateContactAsActive` para manejar de forma más robusta las respuestas de la API de LeadConnector (LynSales).

## Unificación de Centros de Bienestar y Sistema de Notificaciones Enriquecidas

### Added
- **Dashboard de Bienestar Unificado**: Todo el flujo de registro, revisión, aprobación (con modal de bienvenida) y dashboard principal se maneja ahora dentro del widget `wellness-center-widget.js`.
- **Sistema de Notificaciones con Iconos y Links**: Se actualizaron la base de datos y las APIs para soportar notificaciones con iconos visuales y enlaces clicables, mejorando la interacción del usuario.
- **Notificaciones Administrativas**: Alertas automáticas para administradores ante nuevos registros de centros de bienestar (`🏥`) y nuevas apelaciones (`⚖️`).
- **Modal de Bienvenida**: Implementación de un modal de felicitación que aparece automáticamente tras la primera aprobación de un centro.

  - `20260515_create_wellness_center_tables.sql`: Esquema base para centros de bienestar.
  - `20260515_add_welcome_shown_wellness.sql`: Tracking de modales mostrados.
  - `20260515_enhance_notifications.sql`: Columnas `icon` y `link` para notificaciones interactivas.

### Changed
- **APIs de Administración**: Actualización de los endpoints de aprobación/rechazo para inyectar notificaciones enriquecidas al usuario final.
- **Registro de Centros**: El proceso de registro ahora notifica al administrador en tiempo real mediante el dashboard.
- **Widget de Notificaciones**: El componente frontend `notification-bell.js` ahora renderiza iconos y permite navegación vía enlaces dinámicos.

### Fixed
- **Vercel Build**: Se corrigió un error de tipado que causaba fallos en el despliegue de Vercel. Las rutas de API con parámetros dinámicos ahora manejan `params` como una `Promise` (requerimiento de Next.js 15+).
- **Navegación en Webflow**: Se eliminaron dependencias de páginas externas como `/bienestar/estado`, centralizando toda la lógica de estados en el widget principal.
- **Sincronización de Estados**: Mejora en la persistencia de estados de aprobación y apelación entre Memberstack y Supabase.

## Fix: Sincronización Manual de CRM (Resync)

### Fixed
- **Endpoint `/api/admin/members/[id]/sync-crm`**: Se agregó fallback por email cuando la búsqueda por `memberstack_id` falla. Ahora el endpoint intenta:
  1. Buscar usuario por `memberstack_id` (primario)
  2. Si no encuentra, buscar por `email` (fallback desde query param)
  3. Si aún no encuentra, devolver 404 con mensaje claro
- **Botón "Resincronizar CRM"**: Se modificó `MemberDetailModal.tsx` para incluir el `email` del miembro como query parameter en la petición al endpoint, habilitando el fallback.
- **Logs de debug**: Se agregaron console logs detallados en el endpoint para facilitar diagnóstico (`🔍 CRM Debug`).

### Impacto
Los administradores ahora pueden resincronizar manualmente el CRM incluso si el `memberstack_id` tiene discrepancias, mejorando la recuperación de datos y reduciendo soporte técnico.
