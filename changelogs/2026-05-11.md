# Changelog - 2026-05-11

## Resumen del día
Hoy se implementó el flujo completo de **Reembolsos Solidarios (Opción: Manual Asistido)**. Esta mejora permite a los miembros de Club Pata Amiga solicitar reembolsos de gastos médicos proporcionando sus datos bancarios de forma segura, y ofrece a la administración herramientas para gestionar y exportar esta información para su dispersión masiva.

## Cambios Realizados

### Base de Datos (Supabase)
- **feat**: Se creó la migración `20260511_add_bank_details_to_solidarity.sql` para añadir soporte de datos bancarios.
  - Nuevas columnas en `solidarity_requests`: `bank_name`, `bank_clabe` (18 dígitos) y `bank_holder`.

### Widgets del Miembro (Frontend)
- **feat (Widget de Solicitud)**: Actualización de `solidarity-request-form.js`.
  - **Captura Bancaria**: Se integró una nueva sección de "Datos para Reembolso" con diseño brutalista premium siguiendo las guías de Figma.
  - **Límites Dinámicos**: Implementada API `/api/solidarity/balance` para control de topes anuales por mascota.
- **fix (Widget de Configuración de Usuario)**: Actualización de `user-settings-widget.js`.
  - **Sincronización de Fecha de Pago**: Se resolvió un error donde no se mostraba la fecha del siguiente cobro. Ahora el widget consume la API `/api/user/payment-method` para obtener datos precisos de Stripe, garantizando consistencia con el widget de perfil.
  - **Formateo de Fechas**: Se estandarizó el formato de fecha en español ("DD de [Mes] de YYYY") usando el sistema de formateo global del portal.
  - **Gestión de Suscripción**: Se transformó el enlace simple de "Administrar suscripción" en una tarjeta de información detallada que muestra el nombre del plan activo, el costo mensual (moneda local) y la fecha exacta del próximo cobro.
  - **Higiene de UI**: Se ocultó temporalmente la opción de notificaciones por WhatsApp para simplificar la interfaz mientras se finaliza la integración del servicio de mensajería.
  - **Consistencia Visual**: Los nuevos elementos siguen el sistema de diseño del Club (tipografía Outfit, bordes redondeados de 24px y colores de marca).
- **feat (Widget Unificado)**: Actualización de `unified-membership-widget.js`.
  - **Beneficios Dra. PATI**: Se actualizaron los textos del chat con la Dra. PATI en la vista de revisión para ser más directos y claros ("¡No tienes que esperar!", "Acceso al chat para dudas sobre salud, comportamiento o cuidados").
  - **Detalle de Mascota**: Optimización visual del modal de detalles.
    - Se corrigió la alineación del recuadro de "Estatus de tiempo de espera" (eliminando rotación) para un aspecto más limpio.
    - Se reposicionó el recuadro justo debajo de las fotos, eliminando el scroll innecesario para ver el progreso de carencia.
- **UI de Saldos**: Actualización visual del widget para mostrar disponibilidad en tiempo real y bloqueo de beneficios agotados.
  - **Autocompletado Mágico**: Implementada detección automática del banco basada en los primeros 3 dígitos de la CLABE (estándar ABM México), mejorando significativamente la UX y reduciendo errores manuales.
  - **Revelado Progresivo**: La sección bancaria solo se muestra si el usuario selecciona el método "Reembolso".
  - **Validación**: Implementada validación estricta de CLABE (longitud exacta de 18 caracteres y solo números) antes de permitir el envío.
  - **UX**: Limpieza automática de espacios en el campo CLABE.

### Control de Acceso y Roles (Webflow Integration)
- **feat (API de Roles)**: Actualización de `/api/auth/check-role`.
  - Se implementaron cabeceras **CORS** para permitir llamadas seguras desde dominios externos (Webflow).
  - El endpoint ahora soporta pre-vuelo (OPTIONS) y retorna el rol `ambassador` basándose en el estatus real en Supabase.
- **feat (Sincronización Memberstack)**: Creación del endpoint `/api/admin/ambassadors/sync-memberstack`.
  - Herramienta administrativa para sincronizar retroactivamente el estatus de embajador desde Supabase hacia Memberstack.
  - Asegura que el campo `is-ambassador` esté actualizado para todos los miembros aprobados, permitiendo el uso de atributos nativos de Memberstack (`data-ms-content`) en Webflow.
- **fix (Integración de Embajadores)**: Se aseguró que las actualizaciones de estatus (aprobación/rechazo) en el dashboard admin se reflejen instantáneamente en Memberstack, manteniendo la "fuente única de verdad" en Supabase pero con redundancia para performance en el frontend.

### Backend (API Routes)
- **feat**: Actualización del endpoint `POST /api/solidarity/request`.
  - Ahora acepta y persiste los campos `bank_name`, `bank_clabe` y `bank_holder` en la base de datos de Supabase.

### Administración (Dashboard)
- **feat (Detalle de Solicitud)**: Actualización del componente `SolidarityRequestDetail.tsx`.
  - Se añadió un bloque visual con los datos bancarios del usuario para solicitudes de reembolso.
  - **Copiado Rápido**: Implementada funcionalidad para copiar la CLABE al portapapeles con un solo clic.
- **feat (Exportación)**: Actualización del componente `SolidarityDashboard.tsx`.
  - **Exportar CSV**: Se añadió un botón para generar y descargar un archivo CSV con todos los reembolsos aprobados/pendientes que contienen datos bancarios.
  - El formato CSV está optimizado para Excel (evitando notación científica en CLABEs) para facilitar la dispersión masiva vía banca electrónica.
- **fix (Dashboard Crash)**: Resolución de crash crítico ("Cannot read properties of null (reading 'auth')").
  - Implementada programación defensiva con encadenamiento opcional (`optional chaining`) en `MemberDetailModal.tsx` y `RequestsTable.tsx`.
  - Se corrigió el acceso inseguro al objeto de miembro durante la inicialización de estados y ciclos de renderizado, garantizando estabilidad ante datos nulos o cargas parciales.

### Auditoría y Sincronización de Lógica de Carencia
- **feat (Utilidad Centralizada)**: Creación de `src/utils/carencia.utils.ts`.
  - Se centralizaron las reglas de negocio en un solo lugar: 90 días para embajadores, 120/150 días para mascotas adoptadas y 180 días estándar.
- **fix (Admin Approval)**: Actualización de `/api/admin/members/[id]/pets/[petId]/status`.
  - El sistema ahora aplica correctamente el beneficio de 90 días (Referido de Embajador) al aprobar mascotas, resolviendo la inconsistencia previa donde se ignoraba este código.
- **fix (Vet-Bot API)**: Resolución del error crítico 42703 (columna `ambassador_code` inexistente) y sincronización de días de carencia.
  - Mejora del endpoint `/api/integrations/vet-bot/context`: Implementación de metadatos detallados (`daysRemaining`, `label`) y lógica de fallback que consulta la tabla `referrals`.
  - **Corrección de Cálculo**: Se añadió `is_adopted`, `is_mixed_breed` y `created_at` a la consulta de mascotas para asegurar que Dra. PATI calcule correctamente los días transcurridos y restantes.
  - **Nueva Utilidad**: Implementación de `getDaysElapsed` para mostrar el progreso real de la membresía.
  - **Mejora de Payload**: Se añadió `daysElapsed` al objeto `waitingPeriod` y se actualizó el `label` para ser más descriptivo (ej. "6 días transcurridos, 144 días restantes").
  - **Emergency Fix**: Se eliminó la columna `is_mixed` de las consultas explícitas ya que causaba un error 42703 (columna inexistente en la tabla `pets` de producción).
- **fix (Solidarity API)**: Sincronización de `/api/solidarity/request` con la utilidad central y desacoplamiento de la columna `ambassador_code`.
  - Se garantizó que el acceso a fondos solidarios sea 100% coherente con el reglamento del Club incluso ante inconsistencias de esquema.
- **fix (Pets Add API)**: Se añadió protección (`try-catch`) al registrar nuevas mascotas para asegurar la continuidad del servicio durante actualizaciones de base de datos.
- **docs**: Actualización del manual técnico `Documentacion/ENTREGA-API-VET-BOT.md` con los nuevos campos y ejemplos de integración para la agencia del bot.

### Base de Datos (Supabase)
- **feat**: Migración `20260511_add_ambassador_code_to_users.sql` para añadir formalmente la columna `ambassador_code` a `users`.

### Administración de Miembros y Mascotas (Dashboard)
- **feat (Edición de Miembros)**: Implementada la funcionalidad para que administradores editen el email de los miembros directamente desde el modal de detalles.
  - **Sincronización Crítica**: Actualización simultánea en la autenticación de Memberstack y en la tabla `users` de Supabase.
  - **Verificación Automática**: El sistema dispara un correo de verificación a la nueva dirección email inmediatamente después del cambio para validar la propiedad.
- **feat (Edición de Mascotas)**: Habilitada la edición del nombre de mascotas por parte de administradores.
  - **Mapeo Dinámico**: Sincronización con campos personalizados de Memberstack (`pet-X-name`) y la tabla `pets` en Supabase.
- **UI/UX (Edición Discreta)**: Diseño de interfaz tipo "inline edit" con links de "Editar" que se transforman en campos de entrada, manteniendo un dashboard limpio pero funcional.
  - **Refinamiento Estético**: Los botones de edición ahora utilizan tipografía **Outfit de 12px**, color negro con 80% de opacidad y subrayado.
  - **Iconografía de Acción**: Se rediseñaron los iconos de guardar (✓) y cancelar (✕) para ser minimalistas, sin bordes ni fondos pesados, con feedback visual dinámico (hover).
- **Seguridad**: Todas las operaciones administrativas utilizan `service_role` en Supabase para asegurar la persistencia ignorando restricciones de RLS, pero protegidas por validación de sesión de administrador en el backend.

## Impacto Funcional
- **Estabilidad**: Se eliminaron los crashes reportados en el chat de Vet-Bot y el dashboard administrativo.
- **Resiliencia**: El sistema ahora es capaz de manejar inconsistencias entre el código y el esquema de base de datos sin afectar la experiencia del usuario final.
- **Para el Miembro**: Proceso de solicitud de reembolso más claro y profesional, asegurando que su pago llegue a la cuenta correcta.
- **Para el Administrador**: Reducción drástica del tiempo de gestión manual de pagos gracias a la centralización de datos y la exportación masiva para SPEI.
