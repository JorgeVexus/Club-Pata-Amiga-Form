# Changelog - 2026-05-06

## [1.2.3] - 2026-05-06

### 🎨 UI/UX Improvements
- **Notification Icon Refresh**: Replaced the legacy bell emoji (`🔔`) with a custom brand SVG across all notification widgets.
  - Updated `public/widgets/realtime-notifications.js` (Neo-Brutalist version).
  - Updated `public/widgets/notification-bell.js` (Standard version).
- **Transparency & Alignment**: Refined the bell's visual integration:
  - Set background to `transparent` and removed rigid button borders to rely on the SVG's internal circle.
  - Updated icon color to `#FFFFFF` for better visibility on dark/turquoise navbars.
  - Improved vertical alignment using `inline-flex` and `vertical-align: middle`.
  - Adjusted badge positioning for a cleaner, centered look.
- **SVG Optimization**: Implemented the SVG as a constant for maintainability, using `currentColor` to ensure proper color transitions.
- **Enhanced Toast & Modal**: Integrated the new custom icon into real-time toast notifications and the details modal for a consistent visual identity.

### ⚙️ Backend & Logic
- **Verified "Carencia" Logic**: Confirmed pet waiting period progress calculation in `unified-membership-widget.js` follows a 24-hour block system based on registration timestamps.
- **Dashboard Cleanup**: Temporarily hidden the invoice management box in the canceled plan view to streamline the user experience.

### 🚀 Registration Flow V2 Enhancements
- **Label Refinement**: Updated user-facing text in the registration funnel (`Step1Account.tsx`) to improve the "continuation" experience for returning users.
  - Header: "INICIA SESIÓN" -> "CONTINÚA TU REGISTRO".
  - Button: "ENTRAR 🐾" -> "ENTRAR Y CONTINUAR 🐾".
  - Support: "¿Problemas para iniciar sesión?" -> "¿Problemas para registrarte?".
- **Pantalla Intermedia de Redirección**: Se implementó una pantalla de transición ("¡Ya eres parte de la familia!") que se muestra durante 4 segundos antes de redirigir al portal oficial.
    - **Control de Sesión (Testing)**:
        - Se añadió un botón de "Cerrar sesión" en la pantalla de redirección para permitir a los testers cambiar de cuenta sin quedar atrapados en el bucle de redirección.
        - Se añadió un link de logout en la cabecera del formulario (Paso 1) si detecta una sesión activa.
    - **Refuerzo de Detección**: Ahora detecta planes activos mediante `planConnections`, `planIds` y `payment-status` custom field.
- **Salto de Pasos Inteligente (Corregido y Robustecido)**:
    - **Bug Fix**: La detección de mascotas buscaba en campos de la tabla `users` (`pet_name`, `pet_type`) que siempre están vacíos. Las mascotas viven en la tabla separada `pets`.
    - **Solución**: Ahora se usa `getPetsByUserId()` que consulta directamente la tabla `pets` (fuente de verdad). Si el usuario tiene ≥ 1 mascota, se envía al **Paso 3** sin importar los metadatos de Memberstack.
    - **Optimización**: En la carga inicial (`loadSavedState`), las consultas de usuario y mascotas se ejecutan en paralelo con `Promise.all()` para no añadir latencia.
- **Limpieza de UI**:
    - Se eliminó el botón de "Cerrar sesión" redundante que flotaba sobre el formulario, ya que el Navbar ahora maneja esta opción de forma consistente desde el Paso 1.
    - Se restauró el ancho completo (`contentWide`) para el Paso 1.
- **Smart Redirection**: Implemented a security redirect in `NewRegistrationFlow.tsx` for already paid members.
  - If a member with an active plan (Monthly/Annual) attempts to log in through the registration flow, they are automatically redirected to the official login page (`https://www.pataamiga.mx/user/inicio-de-sesion`).
  - **Fix (Reinforcement)**: Made detection more robust by handling case-insensitive status (`active`/`ACTIVE`) and including trial status (`trialing`). Added console debug logs for real-time verification.
  - Added an exception for users who just completed a payment to ensure they can finish Step 4 (Profile) and Step 5 (Pets).
- **QA & Stability**: Verified full production build and type integrity for the updated registration logic.
