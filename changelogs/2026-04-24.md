# Changelog — 2026-04-24

## feat: Habilitación de carga de certificados PDF para mascotas

### Resumen
Se habilitó el soporte para archivos PDF en los flujos de certificación veterinaria de la plataforma, permitiendo a los usuarios subir documentos médicos en un formato más común además de imágenes tradicionales.

---

### Archivos Modificados

#### `src/app/api/upload/pet-photo/route.ts`
- Se actualizó `MAX_FILE_SIZE` de 5MB a **10MB**.
- Se añadió `application/pdf` a la lista de tipos permitidos (`ALLOWED_MIME_TYPES`).
- Se actualizó el mensaje de error para reflejar el soporte de PDF y el nuevo límite de tamaño.

#### `src/app/api/user/upload-pet-photo/route.ts`
- Se sincronizó la validación del lado del servidor para permitir archivos `application/pdf`.
- Se reforzó el límite de tamaño a 10MB para mantener consistencia con la API de carga general.

#### `public/widgets/unified-membership-widget.js`
- Se actualizó la validación del lado del cliente en `uploadPhoto` para permitir archivos de hasta **10MB**.
- Se validó la lógica de previsualización para mostrar un icono de documento (📄) cuando se carga un archivo no-imagen (PDF), evitando errores de renderizado.

#### `public/widgets/pet-cards-widget.js`
- Se añadió validación explícita del lado del cliente en `uploadNewPetPhoto` para tipos de archivo (JPG, PNG, PDF) y tamaño (10MB).
- Se actualizó el límite de tamaño a 10MB.
- Se verificó que los modales de carga manejen correctamente el estado visual durante el procesamiento de PDFs.

- Integración con el sistema de notificaciones para alertar a los administradores sobre nuevos mensajes.

## feat: Estandarización de interfaz de comunicación y acceso global a Chat

### Resumen
Se estandarizó el acceso al sistema de comunicación ("Pata Chat") para asegurar que esté disponible consistentemente en todos los estados que requieren interacción (Acción Requerida, Rechazada, Apelada). Ahora, todas estas vistas convergen en un único modal de detalles centralizado.

---

### Archivos Modificados

#### `public/widgets/unified-membership-widget.js`
- **Estandarización de Botones**: Se reemplazaron botones fragmentados (`pata-btn-history`) por el botón unificado `pata-btn-ver-detalles` en las vistas de mascotas apeladas, rechazadas y con acción requerida.
- **Acceso Directo**: Se añadió un botón de "Chat con Soporte" prominente en el estado `action_required` para facilitar la resolución de dudas.
- **Delegación de Eventos Optimizada**: Se actualizó el listener global para usar `data-pet-id`, permitiendo abrir el modal correcto independientemente de la pestaña activa en el dashboard.
- **Consolidación de Modales**: Se eliminó la dependencia de modales de historial secundarios, integrando toda la línea de tiempo y chat interactivo en el modal principal de expediente de la mascota.

---

### Impacto funcional

- **Consistencia de UX**: El usuario ahora tiene una experiencia predecible; cualquier interacción relacionada con el estatus de su mascota abre el mismo expediente detallado con chat.
- **Robustez**: Se eliminaron posibles errores de sincronización de pestañas al usar IDs explícitos en los disparadores de eventos.
- **Accesibilidad**: El chat ya no es exclusivo de un estado específico, permitiendo que el historial de comunicación esté siempre disponible para consulta del miembro.

---

## feat: Unificación total de comunicación y soporte de archivos en Pata Chat

### Resumen
Se simplificó la interfaz de usuario eliminando puntos de acción redundantes y centralizando toda la comunicación (apelaciones y actualizaciones de información) en el sistema de chat interactivo. Además, se habilitó el soporte para adjuntar archivos y fotos directamente desde la interfaz de chat.

---

### Archivos Modificados

#### `public/widgets/unified-membership-widget.js`
- **Simplificación de UI**:
    - Se eliminaron los botones de "Actualizar Información" y "Apelar Solicitud" de los banners de estado `action_required` y `rejected`.
    - Se implementaron botones unificados de acción principal ("Chat y Actualizar" / "Chat y Apelar") que funnelan al usuario directamente al expediente interactivo.
- **Soporte de Archivos en Chat**:
    - Se añadió un botón de adjunto (📎) en el área de entrada del chat.
    - Implementación de `handleChatUpload` para procesar subidas inmediatas a Supabase Storage.
    - Los archivos subidos se envían automáticamente como mensajes de chat con enlaces formateados.
- **Renderizado de Mensajes Enriquecido**:
    - Se mejoró la visualización de las burbujas de chat para detectar links de imágenes y archivos.
    - Las imágenes adjuntas ahora muestran una previsualización directa en el historial del chat.
    - Los documentos adjuntos muestran un icono de archivo con el nombre del documento para facilitar su identificación.

---

### Impacto funcional

- **Fricción Reducida**: Al eliminar botones redundantes, se guía al usuario de manera más clara hacia el canal de comunicación adecuado.
- **Comunicación Enriquecida**: La capacidad de enviar fotos directamente en el hilo de chat permite una resolución más rápida de estados de "Acción Requerida".
- **Contextualidad**: Toda la documentación enviada por el usuario (fotos, certificados) queda ahora registrada cronológicamente en el historial de chat, facilitando la revisión administrativa.

## style: Unificación de Sistema de Diseño y Accesibilidad Pro

### Resumen
Se realizó una auditoría técnica profunda y unificación estética de los widgets de membresía y tarjetas de mascotas. El objetivo principal fue eliminar patrones de diseño genéricos ("AI slop"), centralizar tokens de marca y asegurar el cumplimiento de estándares de accesibilidad (WCAG).

---

### Archivos Modificados

#### `public/widgets/unified-membership-widget.js`
- **Tokens de Diseño**: Centralización de colores y dimensiones en variables CSS (`:root`).
- **Eliminación de Anti-patrones**: Se eliminó el borde lateral acentuado (BAN 1) en la sección de beneficios por ser un recurso sobreutilizado.
- **Accesibilidad**: Añadidos roles ARIA (`role="dialog"`, `aria-modal="true"`) y etiquetas descriptivas (`aria-label`) en botones iconográficos.
- **Optimización de Chat**: Ajuste del layout editorial para asegurar que el historial del chat sea siempre visible mediante el uso de `clamp()` y scroll independiente.
- **Rendimiento**: Implementación de `loading="lazy"` en toda la galería de fotos del modal.

#### `public/widgets/pet-cards-widget.js`
- **Consistencia de Marca**: Sincronización completa con el sistema de variables del widget unificado (bordes de 4px, paleta turquesa/naranja corregida).
- **Accesibilidad en Formularios**: Vinculación explícita de labels e inputs mediante IDs y atributos `for`. Añadidos indicadores de progreso accesibles en el registro multi-paso.
- **Refinamiento Visual**: Simplificación de tarjetas de mascotas, eliminando sombras difusas y gradientes genéricos de IA para adoptar una estética de bordes definidos y limpia.
- **Responsividad**: Optimización del grid de fotos y formularios para dispositivos móviles ultra-pequeños.

---

### Impacto funcional

- **Consistencia Visual**: Los dos widgets principales ahora se ven y se sienten como parte de la misma aplicación premium.
- **Inclusión**: Los usuarios con tecnologías de asistencia ahora pueden navegar y completar el registro de mascotas sin barreras.
- **Mantenibilidad**: Los cambios de marca ahora pueden realizarse modificando solo 4 variables CSS en lugar de cientos de líneas de código JS.
- **Velocidad de Carga**: Mejora en el tiempo de interacción inicial gracias a la carga diferida de imágenes pesadas.
