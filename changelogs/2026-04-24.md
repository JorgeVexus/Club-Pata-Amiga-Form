# Changelog — 2026-04-24

## feat: Habilitación de carga de certificados PDF para mascotas

### Resumen
Se habilitó el soporte para archivos PDF en los flujos de certificación veterinaria de la plataforma, permitiendo a los usuarios subir documentos médicos en un formato más común además de imágenes tradicionales.

---

### Archivos Modificados

#### `src/app/api/upload/pet-photo/route.ts`
- Se actualizó `MAX_FILE_SIZE` de 5MB a **10MB**.
- Se añadió `application/pdf` a la lista de tipos permitidos (`ALLOWED_MIME_TYPES`).
- Se actualizó el mensaje de error para reflejar el soporte de PDF y el nuevo límite de tamaño.

#### `src/app/api/user/upload-pet-photo/route.ts`
- Se sincronizó la validación del lado del servidor para permitir archivos `application/pdf`.
- Se reforzó el límite de tamaño a 10MB para mantener consistencia con la API de carga general.

#### `public/widgets/unified-membership-widget.js`
- Se actualizó la validación del lado del cliente en `uploadPhoto` para permitir archivos de hasta **10MB**.
- Se validó la lógica de previsualización para mostrar un icono de documento (📄) cuando se carga un archivo no-imagen (PDF), evitando errores de renderizado.

#### `public/widgets/pet-cards-widget.js`
- Se añadió validación explícita del lado del cliente en `uploadNewPetPhoto` para tipos de archivo (JPG, PNG, PDF) y tamaño (10MB).
- Se actualizó el límite de tamaño a 10MB.
- Se verificó que los modales de carga manejen correctamente el estado visual durante el procesamiento de PDFs.

---

## feat: Refinamiento del sistema de comunicación de mascotas (Chat)

### Resumen
Se implementó un sistema de comunicación centralizado y persistente tipo "chat" dentro del widget de membresía unificado. Este sistema permite a los usuarios interactuar directamente con los administradores cuando se requiere acción adicional o hay apelaciones activas, manteniendo un historial transparente por mascota.

---

### Archivos Modificados

#### `public/widgets/unified-membership-widget.js`
- Se integró una interfaz de chat moderna con diseño glassmorphic en el modal de detalles de la mascota.
- Implementación de métodos `renderChatInterface`, `fetchAndRenderChat` y `handleSendMessage`.
- Lógica de activación contextual: el área de entrada de mensajes solo se activa si la mascota está en estado `action_required`, `rejected` o `appealed`.
- Clasificación de mensajes por tipo: Administrador, Usuario y Sistema.

#### `src/app/api/user/appeal-history/route.ts`
- Se expandió el mapeo de iconos y tipos de logs para soportar `user_message` y `admin_message`.

#### `src/app/api/user/chat/send/route.ts` [NEW]
- Nuevo endpoint para procesar el envío de mensajes desde el dashboard del usuario.
- Integración con el sistema de notificaciones para alertar a los administradores sobre nuevos mensajes.

---

### Impacto funcional

- **Centralización**: Se eliminan las solicitudes de información fragmentadas, consolidando todo el diálogo en un solo canal por mascota.
- **Transparencia**: El usuario tiene visibilidad total del historial de su registro, incluyendo cambios de estado automáticos y mensajes humanos.
- **Eficiencia**: Reduce la fricción en el proceso de aprobación al permitir respuestas rápidas a solicitudes administrativas desde el mismo widget.
- **Estética Editorial**: El diseño de las burbujas de chat sigue la identidad visual premium de Pata Amiga, mejorando la percepción de calidad del servicio.
