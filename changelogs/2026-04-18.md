# Changelog - 2026-04-18

## Resumen de Cambios
1. **Sincronización Vet-Bot**: Sincronización integral de los datos de edad y tipo de mascota entre Supabase y el API del Vet-Bot.
2. **Deprecación RUAC**: Eliminación completa del sistema RUAC y simplificación de los periodos de carencia a un esquema de 90/180 días.

## Cambios Técnicos (Vet-Bot Sync)

### Backend (Server Actions)
- **`src/app/actions/user.actions.ts` [MODIFY]**: 
    - Actualizada la función `registerPetsInSupabase` para poblar automáticamente la columna `age` (texto legible) durante el registro.
    - Implementada lógica de pluralización dinámica para el texto de edad ("año" vs "años", "mes" vs "meses").

### API Integrations
- **`src/app/api/integrations/vet-bot/context/route.ts` & `src/app/api/integrations/vet-bot/verify-code/route.ts` [MODIFY]**:
    - **Sincronización y Paridad**: Ambos endpoints del Vet-Bot (Contexto y Verificación por Código) ahora comparten la misma lógica de negocio robusta.
    - **Optimización de Consulta**: Se añadieron las columnas `pet_type`, `age_value` y `age_unit` al SELECT de la tabla `pets`.
    - **Mapeo Robusto de Tipo**: Ahora utiliza el campo estructurado `pet_type` de la base de datos en lugar de inferirlo de la raza, eliminando errores de identificación de especie.
    - **Fallback de Edad**: Implementada lógica de construcción dinámica de edad. Si el campo `age` está vacío (registros antiguos), el API genera el texto a partir de los valores numéricos para asegurar que el bot siempre reciba la edad.

### Frontend UI
- **`src/components/PetRegistrationForm/PetRegistrationForm.tsx` [MODIFY]**:
    - Corregido el mapeo inicial de mascotas existentes para usar `pet_type` de la base de datos.
    - Implementada la reconstrucción de la clave de edad (ej: `4-6-meses`, `2-años`) al cargar datos de Supabase, asegurando que los componentes `Select` de la interfaz reflejen el valor correcto incluso para registros previos a esta actualización.

### Mantenimiento y Datos
- **`scripts/sync-pets-data.js` [NEW]**:
    - Desarrollado script de utilidad para sincronización masiva de datos.
    - **Ejecución Realizada**: Se actualizaron exitosamente **54 registros** de mascotas en Supabase, poblando la columna `age` que anteriormente contenía valores nulos.

## Cambios Técnicos (Deprecación RUAC)

### Business Logic & Services
- **`src/services/pet.service.ts` [MODIFY]**: Se eliminó el beneficio de 120 días para mascotas mestizas/RUAC. Ahora solo existen los niveles de 90 días (Adopción/Referido) y 180 días (Estándar).

### Webflow Components
- **`webflow-components/pet-cards-section.html` [MODIFY]**: Eliminado el campo de entrada de RUAC y la lógica de cálculo de carencia asociada.
- **`waiting-period-panel.js` & `.html` [MODIFY]**: Limpieza de referencias a RUAC en mensajes informativos y etiquetas del dashboard.
- **`WEBFLOW-WAITING-PERIOD-GUIDE.md` & `WEBFLOW-DASHBOARD-GUIDE.md` [MODIFY]**: Actualización de guías para reflejar la eliminación del campo.

### Legal & Documentation
- **`src/data/legal-terms.ts` [MODIFY]**: Eliminación de todas las cláusulas, campos de datos y referencias al RUAC en los términos y condiciones.
- **`MEMBERSTACK-FIELDS.md` [MODIFY]**: Eliminación de la documentación de los campos `pet-X-ruac`.
- **`Documentacion/` [CLEANUP]**: Limpieza de menciones a RUAC en archivos de observaciones y documentos históricos.

### Database
- **`supabase/migrations/20260418_remove_ruac_from_pets.sql` [NEW]**: Script para eliminar físicamente la columna `ruac` de la tabla `pets`.

## Impacto Funcional
- **RUAC**: El sistema ya no solicita ni procesa el RUAC en ninguna etapa.
- **Transparencia**: El cálculo del periodo de carencia es ahora más directo (90 o 180 días), eliminando confusiones sobre requisitos de validación externa.
- **Vet-Bot**: El asistente virtual ahora reconoce correctamente la especie (perro/gato) y la edad de todas las mascotas, mejorando la precisión del consejo médico.
- **Consistencia de Datos**: Se eliminó la divergencia entre los campos numéricos y de texto en la tabla de mascotas.
