# Changelog - 2026-05-09

## 🐾 Fondo Solidario: Sistema de Interacción y Gestión Administrativa

Se ha implementado un sistema robusto de comunicación bidireccional para el Fondo Solidario, permitiendo que los administradores interactúen directamente con los miembros sobre casos médicos específicos.

### 🚀 Nuevas Funcionalidades
- **Chat de Seguimiento**: Implementación de `solidarity_messages` para un historial persistente de interacción entre Admin y Miembro.
- **Widget de Detalle (Usuario)**: Nuevo widget `solidarity-request-detail.js` para Webflow que permite ver el estatus del caso y chatear con el admin.
- **Dashboard Admin Solidario**: Panel completo de gestión (`SolidarityDashboard.tsx`) con filtros por estado.
- **Detalle Administrativo**: Modal de gestión con chat integrado y controles de estatus para el comité.

### 🛠️ Mejoras Técnicas
- **Soporte Next.js 15**: Corrección de parámetros asíncronos (`params`) en todas las API routes del Fondo Solidario.
- **Saneamiento de Formulario**:
    - Campo de CP restringido a 5 dígitos.
    - Atributo `inputmode="decimal"` en campos de monto para mejor UX en móviles.
    - Nuevo campo obligatorio de subida de **Factura/Comprobante** para reembolsos.
- **Deep Linking**: Las notificaciones del sistema ahora redirigen directamente a la página de detalle de la solicitud específica.
- **Seguridad RLS**: Implementación de políticas Row Level Security en `solidarity_messages` para asegurar la privacidad de los casos.

## 🐾 Fondo Solidario: Validación Senior y Estabilización de Admin

Se ha completado la fase de validación documental para mascotas Senior y se ha corregido un error crítico en el panel administrativo que impedía la visualización de solicitudes.

### 🚀 Nuevas Funcionalidades
- **Validación Senior**: El formulario de solicitud ahora detecta automáticamente mascotas de 10+ años y exige la subida de un **Certificado Médico Senior** si no existe uno previo.
- **Enriquecimiento de API**: El endpoint `/api/solidarity/stats` ahora devuelve metadatos de elegibilidad calculados y banderas de requerimiento senior.
- **Galería de Documentos (Admin)**: El detalle de la solicitud en el dashboard administrativo ahora muestra de forma explícita todos los documentos adjuntos (Certificado Senior, Evidencias, Recetas).

### 🐞 Correcciones de Errores
- **Crash de Dashboard Admin**: Se corrigió un error de `TypeError: Cannot read properties of undefined (reading 'length')` causado por un mapeo incorrecto de propiedades en el componente `SolidarityDashboard.tsx`.
- **Carencia Dashboard**: Se optimizó la lógica de `calculateCarencia` en el widget de dashboard para priorizar la fecha de fin calculada por el servidor, garantizando consistencia.
- **Acceso al Fondo**: Se endureció la validación para asegurar que sólo mascotas con estatus `approved` y carencia finalizada vean el fondo como disponible.

### 📂 Archivos Modificados
- `src/app/api/solidarity/stats/route.ts`: Lógica de elegibilidad y senior flags.
- `public/widgets/solidarity-request-form.js`: UI condicional para certificados senior.
- `public/widgets/solidarity-dashboard.js`: Refactor de lógica de carencia y acceso.
- `src/components/Admin/Solidarity/SolidarityDashboard.tsx`: Fix de acceso a datos.
- `src/components/Admin/Solidarity/SolidarityRequestDetail.tsx`: Galería de documentos.

## 🐾 Admin Dashboard: Optimización de Navegación y Métricas Dinámicas

Se han refinado las interfaces administrativas para el Fondo Solidario, corrigiendo errores de navegación y mejorando la visualización de datos estadísticos para una mejor toma de decisiones.

### 🚀 Mejoras en la Interfaz
- **Navegación Selectiva en Sidebar**: Se corrigió el resaltado de la sub-navegación del Fondo Solidario. Ahora el sistema distingue entre solicitudes nuevas, aprobadas, rechazadas y en proceso, eliminando el error de selección múltiple.
- **Métricas Contextuales Dinámicas**:
    - Se implementó un sistema de cambio de contexto en las tarjetas superiores (`MetricCards`). Cuando el administrador está en el Fondo Solidario, la tarjeta de **"Miembros"** cambia automáticamente a **"Total Solicitudes"**, mostrando la métrica relevante para esa sección.
    - Se añadió un **Grid de Estadísticas** específico dentro del `SolidarityDashboard` con contadores globales de solicitudes por estado (Nuevas, Activas, Rechazadas).
- **Consistencia de Estados**: Se estandarizaron los identificadores de estado (`new`, `in_review`) en el sistema de tipos de TypeScript y en las etiquetas de la UI, asegurando una visualización coherente en todo el panel.

### 🛠️ Mejoras Técnicas
- **Soporte de Sub-Filtros**: El `AdminDashboard` ahora gestiona de forma nativa el estado de sub-filtros, permitiendo redirecciones precisas desde notificaciones hacia estados específicos del fondo.
- **Validación de Tipos (QA)**: Refactorización de interfaces en `admin.types.ts` para incluir los nuevos estados y mapeos de colores, garantizando estabilidad en el build de producción.

### 📂 Archivos Modificados
- `src/components/Admin/AdminDashboard.tsx`: Lógica de filtrado y props.
- `src/components/Admin/Sidebar.tsx`: Sistema de resaltado de sub-ítems.
- `src/components/Admin/Solidarity/SolidarityDashboard.tsx`: Integración de Grid de estadísticas y filtros.
- `src/components/Admin/MetricCards.tsx`: Lógica de etiquetas dinámicas.
- `src/types/admin.types.ts`: Estandarización de estados y etiquetas.

## 🐾 Fondo Solidario: Refinamiento de Detalle y Evidencia (Figma Alignment)

Se ha completado la fase de refinamiento visual y funcional del detalle de solicitudes, alineando la interfaz con las especificaciones de Figma y optimizando la visualización de la información de mascotas y evidencia médica.

### 🎨 Mejoras en la Interfaz (Figma)
- **Alineación con Figma (Fase G)**:
    - Implementación de **Badges horizontales** triples (Tipo Solicitud, Tipo de Apoyo, Estado) con iconografía específica.
    - Diseño de tarjetas blancas premium para la sección de meta-información y detalles del caso.
    - Tipografía optimizada con `Fraiche` para títulos y `Outfit` (18px) para descripciones y metadatos.
- **Iconografía Contextual**:
    - Se integraron los nuevos iconos `solicitud-clinica.svg` y `solicitud-fecha.svg` para una mejor jerarquía visual en los detalles de la clínica y fecha.

### 🚀 Nuevas Funcionalidades y Mejoras de Datos
- **Mapeo de Evidencia Inteligente**:
    - La galería de documentos ahora muestra etiquetas descriptivas según el tipo de archivo (ej. "Foto Evidencia", "Informe / Receta", "Comprobante / Ticket").
    - Código de colores dinámico por tipo de archivo para una navegación rápida.
- **Información Financiera**: Se añadió la visualización de **Monto Solicitado** y **Monto Total Pagado** directamente en el detalle de la solicitud.
- **Optimización de Ficha de Mascota**:
    - Se eliminaron campos redundantes como "Tamaño" y "Peso" que no se solicitan en el registro.
    - Se agregaron campos reales de la base de datos: **Especie**, **Sexo** y **Color**, asegurando que la información mostrada sea 100% verídica.
- **Sincronización de Base de Datos**: Se actualizaron los mapeos de campos para usar `case_title` y `case_description`, garantizando la persistencia de los datos del formulario.

### 🛠️ Mejoras Técnicas
- **Deep Linking en Notificaciones**: Se verificó la integración del sistema de notificaciones para que los enlaces lleven directamente a la solicitud correspondiente en el dashboard admin.
- **Mock Data para Previsualización**: Actualización del sistema de datos de prueba para permitir tests de diseño sin depender de la API.
