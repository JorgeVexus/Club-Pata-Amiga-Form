# Changelog - 2026-07-07

## fix: resolve reload session loss via session-only cookie guard

**Commit:** `f4a739a`

Se corrigió el cierre de sesión automático que ocurría involuntariamente al recargar la página o al navegar por las secciones del sitio web de Webflow o la aplicación Next.js.

### Cambios principales
- **Cookie de Sesión Centinela**: Se implementó una cookie de sesión centinela (`pata_session_active=true` sin expiración fija) que expira automáticamente al cerrar el navegador.
- **Eliminación del proxy de `Storage.prototype`**: Se removió el interceptor global que redirigía `localStorage` hacia `sessionStorage`. Esto restaura el comportamiento nativo del almacenamiento para Memberstack y evita las condiciones de carrera e inconsistencias durante la recarga.
- **Guard de Sesión en Carga**: Al cargar la página, si la cookie no existe, limpia los tokens de `localStorage` real y crea la cookie. Si la cookie ya existe (indica que el navegador sigue abierto), no limpia nada, permitiendo que Memberstack lea los datos persistentes de forma nativa.
- **Widgets de Webflow y Next.js**: Homologada la lógica en `src/app/layout.tsx` y en los tres widgets externos:
  - `public/widgets/ambassador-widget.js`
  - `public/widgets/unified-membership-widget.js`
  - `public/widgets/wellness-center-widget.js`
- **Documentación**: Creado el plan de trabajo en `Documentacion/planeacion/2026-07-07-correccion-cierre-sesion-temporal.md`.

### Impacto funcional
- Los usuarios ya no sufrirán de cierres de sesión involuntarios al recargar la página o navegar por las secciones de Webflow.
- Se mantiene con total seguridad la expiración y cierre automático de la sesión cuando el navegador se cierra por completo.

- `npm run lint` ejecutado y verificado (0 errores en el código nuevo).
- `npm run build` compilado exitosamente.

## feat: adapt user profile widget to support wellness center role

**Commit:** `69fea50`

Se adaptó el widget de perfil del usuario (`user-profile-widget.js`) para que soporte de forma dinámica e inteligente el rol de Centro de Bienestar (Wellness Center), adaptando la interfaz a sus datos y sucursales empresariales.

### Cambios principales
- **Detección de Rol**: Se agregó la llamada a `/api/wellness/me` en paralelo para verificar si el usuario logueado posee el rol de centro de bienestar.
- **Sección 1 Adaptable (Datos del Centro)**: Se omiten campos personales (como fecha de nacimiento) para mostrar la información comercial: nombre del establecimiento, correo, teléfono de contacto, dirección principal y especialidades/servicios.
- **Sección 2 Removida (Membresías)**: Se oculta por completo el panel de membresía de Stripe (costo, método de pago y opción de cancelación) al ser irrelevante para centros de bienestar.
- **Sección 3 Adaptada (Roles y Sucursales)**: Se lista el rol como "Establecimiento Aliado" de tipo "Centro de Bienestar" e inyecta la lista dinámica de todas las sucursales adicionales registradas (nombre, dirección y teléfono), ocultando la sección de mascotas y embajadores.
- **Formulario de Edición y Cargas**: Se bifurca el modal de edición de perfil para usar un formulario empresarial que conecta con `/api/wellness/update`, y se redirige la subida de foto de perfil/logo a la API `/api/upload/wellness-logo`.
- **Vista Previa**: Se inyectó soporte en la página de preview local (`profile-widget-preview.html`) con un botón interactivo para probar interactivamente el rol de "Centro Wellness".

### Impacto funcional
- Los establecimientos aliados (centros de bienestar) ahora cuentan con un perfil adaptado a su negocio, donde pueden revisar y actualizar sus datos y consultar sus sucursales directamente en Webflow.
- Se mantiene una experiencia limpia y adaptativa en el mismo widget de perfil.
