Reestructuración Completa del Flujo de Registro v2.0

Viernes, 27 de Febrero de 2026
v4.0.0
🔄 Nuevo Flujo de Registro (5 Pasos)
  • REESTRUCTURE
    Flujo optimizado de 5 pasos: Reducción de ~12 clicks a solo 4-5 clicks antes del pago. Nuevo orden:
    1. Paso 1: Cuenta (email, contraseña)
    2. Paso 2: Mascota básica (tipo, nombre, edad)
    3. Paso 3: Plan + Términos + Pago
    4. Paso 4: Perfil completo (CURP, dirección, nacionalidad)
    5. Paso 5: Mascota completa (raza, colores, fotos opcionales)
  • UX
    Fotos post-pago: Las fotos de mascota ahora son opcionales en el registro inicial. Los usuarios tienen 15 días para subirlas desde su cuenta después de completar el pago.
  • VALIDATION
    Sin límite de edad: Eliminado el límite de 9 años. Mascotas de 10+ años requieren certificado veterinario (indicado en el proceso).
📍 Integración SEPOMEX (Códigos Postales)
  • API
    API oficial del gobierno: Integración con api-sepomex.datos.gob.mx para autocompletar dirección (estado, municipio, colonia) al ingresar código postal.
  • CACHE
    Sistema de cache de 2 niveles: Memoria (Map) + Supabase (catalog_sepomex) para reducir llamadas al API gubernamental. TTL: 24 horas.
  • UX
    Timeout y fallback: 5 segundos de timeout. Si el servicio falla, el usuario puede ingresar dirección manualmente.
🐾 Selector de Raza Mejorado
  • UI
    Switch Mestizo/Raza: Nuevo diseño con bordes redondeados, gradiente en selección, checkmark visual (✓) y sombras mejoradas para distinguir claramente la opción activa.
  • AUTOCOMPLETE
    Autocomplete de razas: Al seleccionar "Raza", aparece campo de autocompletado con 20 razas populares de perros/gatos. Navegación con teclado (↑↓ Enter) y filtrado en tiempo real.
  • UX
    Label dinámico: El campo de nombre cambia según tipo: "¿Cómo se llama tu michi?" (gato) o "¿Cómo se llama tu peludo?" (perro).
📋 Términos y Condiciones Optimizados
  • UX
    Aceptación con 1 click: El checkbox de términos ahora acepta todos automáticamente sin abrir modal. El modal solo se abre vía "Ver términos y condiciones" para revisión.
  • UI
    Modal de solo lectura: El modal de términos ahora muestra los documentos ya aceptados (checkmarks visuales ✓) sin opción de modificar. Botón único "Entendido".
📄 Componentes y Documentación
  • COMPONENT
    AdoptionSection: Nuevo componente reutilizable AdoptionSection.tsx para historia de adopción. Extracción del paso 5 para uso futuro en dashboard de usuario.
  • DOCS
    Guía de implementación: Creación de GUIA-IMPLEMENTACION-PRODUCCION.md con migraciones SQL, variables de entorno, checklist pre-deploy y troubleshooting SEPOMEX.
  • UI
    Inputs alineados: Todos los campos del paso 4 ahora están en vertical (nombre, apellidos, fecha, nacionalidad, estado, municipio) para consistencia visual.
🔧 Fixes y Mejoras Menores
  • BUGFIX
    Tarjetas de planes: Restaurados estilos CSS faltantes para las tarjetas de selección de plan (Mensual/Anual) que se habían perdido.
  • BUGFIX
    Checkbox duplicado: Eliminado elemento checkmark duplicado en el checkbox de términos que causaba visualización de dos checkboxes.
  • UX
    Mensaje senior: Actualizado texto para mascotas 10+ años: "Tu mascota califica como senior, después necesitaremos conocer el estado de salud de tu peludo".

    AuthRedirector: El componente de redirección de la página principal (/) también verifica postal-code antes de redirigir al dashboard.

    Logout de Google: Al cerrar sesión, se abre una ventana a accounts.google.com/logout para limpiar la sesión de Google y permitir cambiar de cuenta.
